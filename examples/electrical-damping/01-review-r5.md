## 命中的洞

### H1 · [MAJOR] `CRIT-ROBUSTNESS-COARSE`（r5 为补 r4-H1 新造的门）结构性失明 —— 命中 P17④ + P18④（元模式复发）

**位置**（逐字抄写）。`check_analysis.py` 第 1119–1154 行，整道门挂在一个 `elif` 上：

```python
elif isinstance(rs.get("delta_max"), (int, float)):
    dm = float(rs["delta_max"]); br = rs.get("delta_max_bracket")
    if not (... len(br)==2 ...):                    err("CRIT-ROBUSTNESS-COARSE", ...没 bracket)
    elif not (0 <= br[0] <= dm <= br[1]):            err("CRIT-ROBUSTNESS-COARSE", ...不合法)
    elif (br[1]-br[0]) > 0.05*br[1]:                err("CRIT-ROBUSTNESS-COARSE", ...太粗)
```

**为什么崩（两个独立的构造，都用运行代码证明）。**

**① `delta_max=None` ⟹ 整道门被跳过 —— 而扫描范围太小就会这样报。**
`elif isinstance(rs.get("delta_max"), (int, float))` 对 `None` 为假 ⟹ bracket 三连检查**一条都不跑**。
而 `criterion_matrix.py` 的 `_delta_star(lo, hi)`（第 438 行）：`if _correct_survives(hi): return None, None`。
**⟹ 只要扫描上界 `hi` 小于真边界（0.135 mm），就返回 `None`，`verdict` 写成「在扫描的全部 δ 上都不误杀」—— 一句假话，而门一声不吭。**
本轮侥幸安全，只因作者挑的 `hi=0.30 mm` 恰好 > 0.135 mm。**但没有任何东西强制这一点**：`robustness_scan` 的字段是 `{parameter, why, delta_max, delta_max_bracket, verdict}` —— **根本没有「扫到了多远」这个字段**，所以 `delta_max=None` 到底是「真的处处稳健」还是「没扫够远」，门和读者都无从分辨。

> **这就是 P18④ / 教训 11 的原样重演，又升了一层楼**：
> r1 死于 ±0.5 mm 定位 · r2 死于 ±6 mm 拟合区间 · **r4 死于网格间距（H1）· r5 的新门死于「扫描上界」**。
> **「作者挑的扫描端点」这个病，被从「网格间距」赶到了「扫描上界」，而新门在那儿又瞎了。**
> **`--selftest` 第 1785–1788 行把这个跳过`钉成了「预期行为」**（`delta_max=None ⟹ 期望 []`）—— **盲区被写进了自检，等于给它盖了章。**

**② 手写一个窄括号，就把 H1 判死的那个虚报值 `0.17` 原样塞回来。**
门只查三件事：bracket 存在、`0≤lo≤dm≤hi`、相对宽 <5%。**它从不验证这个括号真的是二分出来的、真的括住了真判据的过/死跳变。** 于是：

| 我喂给门的 `robustness_scan` | 门的判定 |
|---|---|
| `delta_max=1.35e-4, bracket=[1.34e-4,1.35e-4]`（真二分） | 无 ERROR ✓（对） |
| **`delta_max=None`（扫描范围太小的伪报）** | **无 ERROR ✗ 门被完全跳过** |
| **`delta_max=1.70e-4, bracket=[1.694e-4,1.70e-4]`（手写窄括号 @ H1 的虚报值）** | **无 ERROR ✗ H1 的 0.17 原样复活** |
| `delta_max=1.7e-4, bracket=[1.0e-4,1.7e-4]`（网格宽括号 41%） | `CRIT-ROBUSTNESS-COARSE`（这个它抓） |

（脚本 `gate_blindness.py`，五行可复现。）**门抓得到「诚实但太粗的网格」，抓不到「精心手写的窄括号」和「压根不给 delta_max」—— 而后两者才是作弊者会走的路。**

**影响**。这道门是 r5 的招牌新功能，是 r4-H1「delta_max 必须二分定出」逼出来的。它**在本轮诚实的实例上判得对**，但作为**要跑遍每一道未来题的 skill 门**：
- 它没堵住它自己号称要堵的洞 —— **一个手写的窄括号能让 `delta_max` 报任意值**；
- 更糟的是 `delta_max=None` 这条逃生舱：下一道题的作者只要把 robustness 扫描的范围挑得不够宽（**而你事先并不知道边界在哪，这是最自然的错误**），就会得到「处处稳健 ✓」+ 绿灯，**哪怕真边界就落在噪声预算里面**。

**Opponent 的杀招**：
> 「你新加的门要求 `delta_max` 二分定出。我给它 `delta_max=None`、verdict 写『处处不误杀』—— 门一声不吭。我再给它 `delta_max=0.17`、括号 `[0.169,0.170]` —— 就是你 r4 判死的那个虚报值 —— 门还是放行。你的门查的是三个数彼此自洽，从没查它们是不是二分跑出来的。**你 selftest 里还专门写了一行说 `None` 就该放行。**」

**可否修**（都得回填进 skill）：
1. `robustness_scan` 增一个**必填**字段 `scan_upper_bound`（扫到多远），且当 `delta_max=None` 时，门必须校验 `scan_upper_bound ≥ k × 协议自报的系统误差`（如 ≥3× 噪声预算）——否则「处处稳健」是不可证伪的。
2. bracket 不能只查宽度：要么让门**自己复跑一次判据**验证 `br[0]` 过、`br[1]` 死（真正的「不可能撒谎」），要么承认它只是自洽性检查、把裁决交回给必须审 `criterion_matrix.py` 的人。
3. 把 `delta_max=None` 的 selftest 用例改成**双向**：既测「真处处稳健 ⟹ 放行」，也测「扫描范围太小的伪 None ⟹ 抓住」。

## ★ 给 skill 的回填建议（**任何题都会踩的坑**）

### 1. ★★★ 一道新造的机械门，必须**双向**自检 —— 「它在诚实实例上放行」不算验证

> **本轮两道新门都在 `--selftest` 全绿、真实工作区 0 ERROR，而我各找到 ≥2 个结构性放行。**
> **原因和判据、和双向表**一模一样**：selftest 只跑了「它该抓的 → 抓到」和「诚实的 → 放行」，没跑「它号称要挡、但换个形状 → 该抓却放行」。**
> **⇒ 这是 P18「双向表是自评」的**再上一层**：`--selftest` 本身也是自评。**
> **每加一道门，selftest 必须包含「盲区探针」**：对 `CRIT-ROBUSTNESS-COARSE` 是 `delta_max=None`+扫描太窄、和手写窄括号；对 `PROSE-FORMULA-GHOST` 是非 target 幽灵、换措辞幽灵。**造不出让新门失明的输入 = 你还没理解它守的是什么。**

### 2. ★★ 「不误杀=None」必须可证伪 —— robustness_scan 要记「扫了多远」

> `delta_max=None` 现在同时是「真处处稳健」和「没扫够远」的编码，而 schema 没有 `scan_upper_bound` 字段。
> **⇒ `robustness_scan` 增必填 `scan_upper_bound`；当 `delta_max=None`，门校验它 ≥ k×(协议自报的系统误差)。** 否则「处处稳健」是不可证伪的 —— 而不可证伪正是这道题反复在抓的东西。

### 3. ★ 机械门守的是「一类」，不是「上一次事故的措辞」

> `PROSE-FORMULA-GHOST` 把 H2 的确切句式抄成了正则（关键词表 + `targets[]` + ≤10 字桥），于是非 target 派生量、换措辞、数字在前 —— 全溜。**这和 P17③「正则写死一种排版」是同一个病，只是犯在审稿人自己的门里。**
> **⇒ 守护范围扩到「任何带 closed_form/numerical_recipe 的具名量 + impact_if_false 里的派生量」；措辞别写死；说不清能覆盖哪一类，就明说「这一类需人读」，不要伪装成「幽灵数都归我管」。**
