## 命中的问题

### H1 · [REVISE] 交互页（+ I-1 静帧）：Γ 抛物线外推出拟合窗，与 results.json 注册值差 ~85% — 命中 D12

- **位置**：`damping-explorer.html` 读出行与包络曲线本身；`I-1.png` 左面板紫色曲线（`z0=10.45 mm`）。
- **物证**（全部可复算）：
  - 源码 `G23(z0,R) = gamma_oc + c2(R)·z0²` 作用于整个滑块域 `z0 ∈ [0,15] mm`（step 0.25），无任何窗口守卫；
    `DATA` 里没有 G(z₀) 表可回退。
  - 截图 `zpk.png`（z0=10.5, R=0）：页面显示 **`Γ(z₀) = 25.3932 1/s`**
    = 0.0413 + 229949.6·(0.0105)² ✓（就是抛物线）。
  - **本次运行的真值**：`targets[γ].value_numeric = 2.1741`（R=20, z₀=z_pk）
    ⟹ 换算 R=0：(2.1741−0.0413)·(23.71/3.71) + 0.0413 ≈ **13.7 s⁻¹**。
    F-2 图上也能直接读出：x 截距 −3.62 Ω、斜率 0.0199 s/Ω ⟹ 1/(γ−γ_oc)|_{R=0} = 0.072 s ⟹ 13.9 s⁻¹。
  - **25.39 vs 13.7 = +85%**。R=20 档同病：页面会显示 4.01，注册值 2.174（+84%）。
    交叉验证：[G'(0)·z_pk / G(z_pk)]² = (103.55×0.0106/0.8145)² = 1.82 —— 同一个数量级，铁定。
  - 而 F-1 自己的脚注写着 **「parabola fitted on |z₀| ≤ 4 mm」** —— 页面把它用到了 2.6 倍远处。
  - 同页的 regime 边界（`nu_boundary` 表）是从真 G(z₀) 烘焙的、在 z_pk 处正确地拐头 ——
    **同一页面里边界用真物理、Γ 读数用外推抛物线**，内部自相矛盾。
- **为什么严重**：Physics Fight 上对手第一件事就是把滑块拖到 z_pk（F-1/F-2 都把它当招牌）。
  页面在那里给出的阻尼率、画出的包络斜率都错 ~1.85×，且无任何警示。
  I-1 caption 的 provenance 声明（「页面只求值契约闭式 ⟹ 不可能与 results.json 打架」）被当场证伪 ——
  **求值域越界本身就是一次「与 results.json 打架」**。
- **怎么修**（三选一，都保住「no client-side physics」）：
  1. **烘焙 γ(z₀; R) 表**（与 `nu_boundary` 同一张 40 点网格，γ = γ_oc + G(z₀)²/[2M_eff(R+R_c)] 由跑分产出），
     `G23` 改查表插值；抛物线只留在 |z₀| ≤ 4 mm 或画成虚线对照。
  2. 滑块钳到 [0, 4] mm + 单独一个「z₀ = z_pk」preset（用烘焙的 γ(z_pk)）。
  3. 至少：|z₀| > 4 mm 时读出行变灰并加 「parabola window exceeded — see F-1 footnote」。
  修完**重渲 I-1.png、重截三张截图**；`results.json` 里 I-1 的 caption 那句「不可能打架」要改写（Skill 2 的台账）。

### H2 · [REVISE] F-1 左面板：图例把 G(z) 的整个负峰压掉 — 命中 D1/D4

- **位置**：`F-1.png` 左面板，图例框（两行长标签）右段覆盖 z ≈ +4…+17 mm、G ≈ −0.5…−0.9 Wb/m 区域。

  或把两行标签缩短（"Model-0 (eq. 6)" / "Model-2 (eq. 27)"）后放上右角。不许再压任何数据点。

### H3 · [REVISE] F-3(b)：γ_oc 标签骑在结论线上，埋掉两个数据点的一半 — 命中 D1
