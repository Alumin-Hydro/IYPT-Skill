# IYPT 物理分析

## 这个 skill 到底在解决什么

IYPT 的题目是**故意欠定的**：

> "A cylindrical magnet is dropped into a vertical conducting pipe. Investigate the motion of the magnet and how the terminal velocity depends on relevant parameters."

不给磁体尺寸、不给管材、不给壁厚、不说 "relevant parameters" 是哪些、不说扫到什么范围。

**直接开始建模，必然出三种病**：隐式假设（用了"均匀磁化"却没说出口）、口头忽略（"空气阻力可忽略"但没给数字）、脆弱结论（结论悬在一条没验证过的假设上）。这三种病，在 Physics Fight 上都是 Opponent 的一枪毙命。

所以**这个 skill 的核心不是"解题"，是"把题目补全成一个可证伪的物理问题"**，然后从第一性原理推导它，再让一个看不到你思考过程的审稿人反复攻击它。

## 六件真正改变结果的事

0. **★ 任务是「挖」出来的，不是「读」出来的。** 题面是一段话，真正的任务藏在措辞里——"under certain conditions" 是一条任务（找条件边界），"other interesting behaviour" 是一条任务（模式分类 → 混沌）。**挖不出来，你就在答一道自己编的、更容易的题。** 而且**因变量不只是"数"，还有"模式"**——这是 IYPT 的分水岭。（Stage 0.5）
1. **设定 (Setting) 和假设 (Assumption) 必须分开记。** 设定是你选的实验条件（磁体用 10 mm 的），选了就没错。假设是你对物理的简化（把它当点偶极子），可能错。**绝大多数翻车来自把后者当成了前者**——因为你根本没意识到自己做了一个假设。
2. **判据必须写成不等式。** "点偶极子近似通常成立" = 没写。"$r \gg L$，而实测 $r/L = 0.8$" = 你发现自己错了。**判据的价值在于它能被代入数字然后打你的脸。**
3. **每个"忽略"必须以一个数字结束。** 而且要在**扫描范围的两端**都算——基准点可忽略、扫到另一端变成主导，是"曲线右半段整段是错的"的根源。
4. **必须有一个零自由参数的预测。** 幂律的**指数**、无量纲组的**临界值**、坍缩曲线——这些是模型可以被杀死的地方，也是分数所在。"模型和数据吻合得很好"（用了 3 个拟合参数）没有任何证据价值。
5. **审稿人不能看到你的推理过程。** 一长串看起来很努力的推导，读起来就像证据。fresh context，只给清洗过的结论。

## 允许联网

允许 WebSearch / WebFetch 查文献、物性参数、已知模型。**但**：

- 每条引用要给出处，写进「文献」一节；
- **严格区分"文献结论"和"自己推导"**——混淆两者是审稿模式 P15；
- 引用一个公式，要核对**它的适用条件**、它的符号约定、它的单位制（高斯制和 SI 制的电磁公式差 $4\pi$ 和 $c$）；
- 文献能给你参数和思路，**不能替你推导**。关键步骤必须自己推出来。

# 工作流

先建工作区 `iypt/<problem-slug>/`（slug 用英文小写连字符）。产物路径见 `docs/pipeline.md`。

## Stage 0 · 题型判定

读 `references/iypt-context.md`。四种题型，**重心完全不同**：

| 题型 | 重心 |
|---|---|
| 现象解释型 ("Explain why…") | **机制预算**——证明是这个机制，排除其他候选 |
| 依赖关系探究型 ("How does Y depend on…") | **量纲分析 + 参数扫描**——最常见的题型 |
| 装置优化型 ("Maximize…") | **权衡关系**——什么增大了、什么必然减小了 |
| 测量方法型 ("Determine X using…") | **误差分析**——灵敏度、系统误差、精度极限 |

题型判错，重心就放错了。"解释为什么"的题交一堆扫描图，是跑题。

把原题**逐字**抄进 `00-problem.md`，写上题型判定。

## ★ Stage 0.5 · 任务挖掘 + 物理本质

**读 `references/task-excavation.md`。这一步是新加的，而它是最高杠杆的一步。**

### 为什么

**IYPT 的题面是一段话，不是一张任务清单。** 真正的任务全藏在措辞里。

> **磁铁加速器**（真题）："…this 'vehicle' will accelerate over the rows of magnets **under certain conditions**. **Investigate** the phenomenon."
>
> 拿分的报告从这句话里挖出了**三条**任务：
> 1. 指出磁铁加速的**物理本质** ← "Investigate the phenomenon"
> 2. 探究末速度/加速度与参数的关系 ← "accelerate"
> 3. **探究有效加速的条件** ← **"under certain conditions"**
>
> **第 3 条是从一个介词短语挖出来的**，而它是最难、最值钱的一条——它的答案是一张**参数空间相图**（哪片区域会加速、哪片不会），不是一条曲线。

> **磁力牛顿摆**（真题）："The new cradle can act in a similar way to a regular cradle, but **can also exhibit other interesting behaviour**."
>
> 从 "other interesting behaviour" 挖出「**运动模式的分类**」，最后落到**混沌**、**Lyapunov 指数**、**频谱从离散峰展宽为连续宽带**。
>
> **而「混沌」两个字，题面里一个都没有。**

**挖不出来 = 你在答一道自己编的、更容易的题。而 Opponent 会拿真正的那道题问你。**

### 三个动作

**① 逐词追问题面。** 对照 `task-excavation.md` 的映射表：

| 题面里的词 | 挖出来的任务 |
|---|---|
| "Investigate the phenomenon" | **指出物理本质** |
| **"under certain conditions"** / "can" | **找出条件边界** → 参数空间相图 |
| "other **interesting behaviour**" | **模式分类 + 模式转变** → 分岔 / 混沌 |
| "**relevant** parameters" | 哪些相关、哪些**不**相关，**都要给理由** |
| "similar to X **but also**" | **对照**：哪里像、哪里不像、为什么 |

> **铁律：题面里每一个限定词都要被追问一次。漏掉一个限定词 = 漏掉一条任务 = Opponent 的第一个问题。**

**② 变量归纳。** 自变量按**四类**过一遍防漏（系统 / 构型 / 初始条件 / 环境）；因变量分**三层**：

| 层 | 例 |
|---|---|
| 标量 | 终速、频率、幅度 |
| 函数 | $x(t)$、$F(d)$、$U(x)$ |
| **★ 模式** | 周期 / 准周期 / **混沌**；加速 / 减速 / **卡住**；单稳 / 多稳 |

> **第三层是 IYPT 的分水岭。** 新手把 "study the movement" 理解成"测 $x(t)$ 然后拟合"；
> **高分答案回答的是：有哪几种运动模式？它们怎么随参数转变？边界在哪？**

**③ 物理本质：一句话。**

格式：**「X 通过 Y 转换 / 耦合 / 竞争为 Z 的〈某类〉系统」**

| 题 | 物理本质 |
|---|---|
| 磁铁加速器 | **"磁势能向动能的非对称转换"** |
| 磁力牛顿摆 | **"在重力约束下、通过非接触磁排斥实现耦合的非线性多体动力系统"** |
| 磁刹车 | **"重力驱动与涡流耗散的平衡；驱动 ∝ 常数、耗散 ∝ $v$，故必然趋于终速"** |

> **这句话的作用不是修辞，是骨架 —— 它决定了你要画哪些图。**
>
> - "**非对称转换**" ⇒ 必须画**势能景观** $U(x)$
> - "**非线性多体耦合**" ⇒ 必须画**相图、频谱、Lyapunov**
> - "**驱动与耗散的平衡**" ⇒ 必须画**力平衡、弛豫时间、标度律**
>
> **说不出这句话 = 你还没懂 = 你不知道该画什么图。**

### 产出

写进 `00-problem.md`，并落进 `model-spec.json` 的 `tasks[]` / `essence` / `variables`。

每条任务必须带 **`excavated_from`：题面里的哪个词（逐字）**。
> **这一条是防自欺的**：挖不出出处的"任务"多半是你自己编的；而题面里**没被任何任务引用的限定词**，就是你**漏掉的**任务。`check_analysis.py` 机械检查这一条。

## Stage 1 · 设定书 (Specification Sheet)

**把题目没给的条件全部显式定死。** 写进 `00-problem.md`，每条编号 `S-n`：几何与尺度、材料与物性、驱动与初始条件、**参数扫描范围**、环境（温度/重力/空气）、观测量与精度。

每条写：**取值 + 理由 + 结论对它敏不敏感**。

**扫描范围的选择是得分点，认真选**：好的范围能同时展示两个不同机制主导的区间，让曲线上出现转折/临界行为。随便选一组参数，就把这分扔了。

> 这一步不是填空，**是在设计一个实验**。

## Stage 2 · 假设台账 (Assumption Ledger)

读 `references/assumption-catalog.md`——对着那份清单过一遍，**找出你没意识到自己在用的假设**。最危险的假设永远是那些你觉得"这还用说吗"的。

每条 `A-n` 写全五项：

1. **陈述**（"磁体均匀磁化，远场可等效为点偶极子"）
2. **成立判据 —— 必须是不等式**（$r \gg \max(R,\,L)$）
3. **代入检查 —— 代入 Stage 1 的数值，判据到底满不满足**（$r/L = 0.8$ → **不满足！**）
4. **失效边界**（什么时候崩）
5. **分级**：
   - `SAFE`——即使完全不成立，主结论也不变
   - `LOAD-BEARING`——结论依赖它，且它确实成立。**必须写"若它不成立，结论会怎样"**
   - `RISKY`——判据只是勉强满足（差不到一个数量级），或你无法确定。**必须生成一个数值验证任务交给 Skill 2**

> **`RISKY` 不是坏事，`RISKY` 但没标出来才是。** 标出来的 RISKY + 一个验证它的数值实验 = 满分答案；藏起来的 RISKY = 被 Opponent 一击致命。

### ★ 判据里必须列出**所有**相关的尺度

**这是一条真实翻过车的规矩。**

> **血泪教训（magnetic-brake，真实发生）**：点偶极子近似的判据被写成 $a \gg L$（管半径 ≫ 磁体**长度**）。**漏掉了磁体半径 $R$。**
>
> 真正的判据是 $a \gg \max(R,\,L)$ —— 点偶极子要求**整个磁体**相对 $a$ 小，不只是它的长度。
>
> **后果不止于台账**：Stage 5 的极限对拍配方是照着判据写的。判据漏了 $R$，配方就跟着只写"令 $L\to0$"——**照做的话数值解收敛到 3.55 而不是 1**（得到的是个半径 5 mm 的**薄圆盘**，在 $r=6$ mm 处它根本不是点偶极子）。Skill 2 在 Gate 0 上撞了个满怀，然后要花力气才能判断"这是我代码错，还是契约错"。

**写判据时的自检**：

- **把判据里的量推到极限，模型真的退化成你说的那个简单模型了吗？**
- 有没有**别的**尺度，它不小也会让近似崩？（几何尺度、时间尺度、能量尺度）
- **"远场"类的判据尤其危险**：远场要求源的**每一个**尺度都远小于距离。只写一个，一定漏。

## Stage 3 · 量纲分析与标度律

读 `references/modeling-playbook.md` 第一节，做 Buckingham Π。

**在写下任何微分方程之前做这一步。** 它便宜（十分钟）、几乎不会错（只靠量纲）、而且**能在建模前就抓出错误**——详细模型给出的标度律如果和量纲分析矛盾，一定是模型错了。

必须产出：
- 全部 $n-k$ 个无量纲组，**并说出每个的物理意义**（这是哪两个效应的比？是 Re？是 Bond 数？）
- **明确写出"我认为不相关的量及理由"**——这是防漏项的唯一防线
- 预期标度律
- **数据坍缩预言**：不同参数的曲线画成 $\Pi_1$ vs $\Pi_2$ 应当坍缩成一条线。**这张图必须进 `figures[]`**——它是 IYPT 报告里杀伤力最大的一张

### ★ 控制参数独立性：坍缩图的每根「轴」必须真能独立移动那个无量纲组

**这是一条真实翻过车的规矩（fractal-fingers，r1 审稿 MAJOR）。**

一个自变量若通过某个**本构 / 平衡关系**被一个 `derived` 量决定（前沿速度 $U$ 由 Marangoni 定 $U=\Delta\gamma h/\mu_2 R$），**代进无量纲组后它可能精确抵消**——你以为在扫它、其实 $\Pi$ 一动不动。

> **血泪教训**：$\mathrm{Ca}=\mu_2 U/\gamma$，看着 $\mu_2$（稀释度，**题面唯一明给的旋钮**）是独立轴。但 $U\propto1/\mu_2$（Marangoni）⟹ 代入后 **$\mu_2$ 精确消失**，$\mathrm{Ca}=\Delta\gamma h/(R\gamma)$。「扫稀释改 Ca」全错；稀释的**正确**预言是「$\lambda^\ast$ 与稀释近乎无关」——**与原框架相反**。而且**单变量指数也被耦合污染**（$\lambda^\ast\propto h^{1}\to h^{0.5}$、$N\propto R\to\sqrt{R/h}$），因为 $U$ 也依赖 $h$、$R$。

**自检（写坍缩图、写单变量标度律之前做）**：

- **把每个 `derived` 量的关系（$U=\dots$）代进每个无量纲组和每条标度律**，逐一问：
  - 哪个「自变量」**消失了**？——它不是坍缩轴，是相图**别的**轴（失稳存在性 / 时标 / 阈值）。
  - 哪条单变量指数**变了**？——报「固定 $U$ 下 $h^1$」而 $U$ 不能在固定下扫 $h$，是空头支票。
- **能真正沿 $\Pi$ 挪的「干净旋钮」是哪几个？** 坍缩图的 `series` 必须**老实标成它们**，不是随手写 $(U,\mu_2,\gamma,h)$。
- **Model-0 自己输出的坍缩往往是代数恒等式**（恒真、不构成检验）——**真正的检验是实验能否沿 $\Pi$ 挪、量到那个指数**。别把「仿真复现了自己的公式」当成验证。

> **`check_analysis.py` 对这条永远零 ERROR —— 洞在物理耦合里，不在语法里。它靠对抗式审稿（P12「自由参数/独立性」+ P3）抓。这就是审稿存在的理由。**

## Stage 4 · 机制预算 (Mechanism Budget)

对着 `references/modeling-playbook.md` 第三节的候选机制清单，**逐个问"这题里有没有它"**（重力/惯性/弹性/摩擦/黏性/压差/表面张力/浮力/静电/磁/涡流/传导/对流/辐射/相变…）。

排成一张表：机制 | 表达式 | **代入 Stage 1 的具体数值** | 量级 | **与主项之比** | 去向。

**两条硬规则**：

- **"忽略"必须给出数字。** 口头忽略一律视为错误（审稿模式 P2，最高频）。
- **必须做扫描端点检查。** 比值不是一个数，是参数的函数——把它写成函数，代入扫描范围的**两个端点**。一端可忽略不代表另一端可忽略。

还要列出**你考虑过但认为这题里不存在的机制**。空着不写就是漏项。

判定：比值 $\lesssim 10^{-2}$ → 忽略；$10^{-2}$–$10^{-1}$ → correction（进 Model-1）；$\gtrsim 10^{-1}$ → dominant（必须进 Model-0）。

## Stage 5 · 分层推导

**三层。上来就 FEM 硬解，说明你没理解这个现象。**

- **Model-0 · 玩具模型**：只留主导机制，**求出闭式解**。要能手算、能一眼看懂物理。写完之后用一句话说清"到底发生了什么"——说不清楚，说明你还没懂。
- **Model-1 · 修正模型**：把 correction 项加进去，微扰或近似解析。
- **Model-2 · 完整模型**：需要数值 → 写清方程、状态变量、初始条件、**边界条件（写清物理含义——边界条件错是最隐蔽的错误，P6）**、建议数值方法、**以及"数值解在什么极限下应该回到 Model-0"**（这是 Skill 2 验证代码正确性的手段）。

公式统一编号 `\tag{n}`，符号表统一，**不许"可以证明"跳步**——跳掉的那一步就是审稿人要撬的地方。

### ★ 极限对拍配方：写完必须自己在纸上走一遍

`equations[].numerical_notes` 里那个"数值解在什么极限下回到闭式解"，是 Skill 2 的 **Gate 0**——**它是整个下游验证阶梯的地基**。写错了，Skill 2 会在第一道门上撞死，而且分不清是自己代码错还是你的配方错。

**配方必须写全三件事**：

1. **哪些量趋于什么**（$R\to0$，$L\to0$，$w\to0$）
2. **哪些量保持固定**（$m = M_s\pi R^2 L$ 固定 ⟹ $M_s \to M_s/\epsilon^3$）
3. **误差门槛**（"< 0.1%"），以及**误差必须随极限参数单调趋于 0**

**写完，自己走一遍**：

> **把配方里的量真的推到极限，剩下的模型**真的**是那个闭式解吗？**

**最常见的错**（真实发生过）：**只推了判据里显式提到的那个量，漏掉了另一个同样必须变小的尺度。** 见 Stage 2 的「判据里必须列出所有相关的尺度」——**判据漏一个尺度，配方就跟着漏一个。**

> **指纹**：如果 Skill 2 报告"比值收敛到一个**稳定的、但不是 1 的**值"，那就是**极限存在，但取错了极限**。这是配方错，不是代码错。

## Stage 6 · 自洽性检验

交审稿人之前**自己先跑完这六项**（`references/modeling-playbook.md` 第四节）：

1. **量纲齐次性**——逐式检查，每一个编号公式，包括中途引入的常数
2. **极限行为**——参数推到 0 / ∞，是否退化成已知结果？（抓错误效率最高的一项）
3. **对称性与守恒**——耗散系统 $dE/dt \le 0$，**检查符号**
4. **数值可信度**——代入数值算出一个具体的数，物理上可能吗？（磁铁在铜管里如果算出 5 m/s，模型错了）
5. **已知特例**——退化到教科书结果了吗
6. **反向常识**——把结论念出来，听着就不对的时候，通常是真的不对

## Stage 7 · 可证伪预测 + 交接

**这一节是分数所在。** 必须有**不含自由参数**的预测：幂律**指数**、无量纲组的**临界值**、**坍缩曲线**。

"模型和数据吻合很好"不算——如果你有 3 个自由参数而数据只有 2 个特征，那个"吻合"是被拟合出来的，没有证据价值（P12）。

还要写：**"我什么时候会错"**——哪条假设不成立会让哪个预测怎么变。

同时产出两样东西：

- **实验方案**（写进 `01-analysis.md`）：器材、测量方法、**要证伪预测所需的精度**、误差来源、哪些结论仍需真实实验确认。
  > **仿真不能替代实验。** 仿真验证"方程解对了"，实验验证"方程写对了"。只有实验能证伪模型。这一节是给人去做的，不是走过场。
- **`handoff/model-spec.json`**：Skill 2 的唯一输入契约。schema 见 `templates/model-spec.schema.json`。

  硬约束：
  - 每条 `RISKY` 假设必须有对应的 `risky_assumption_checks` 条目（含 `degenerate_signature`）
  - 每张图必须有 `expected_shape`
  - **★ 每条 `task` 必须被至少一张图或一个 target 回答**（`figures[].answers_task`）—— **没有任务归属的图是装饰；没有图回答的任务是空头支票**
  - **★ `essence.one_sentence` 里的每个关键词，都要有一张对应的图**

正文按 `templates/analysis-template.md` 的骨架写到 `01-analysis.md`。

### ★ 图的骨架由「物理本质」决定，不是随便挑几个参数扫一扫

**回到 Stage 0.5 写下的那句话，逐词兑现：**

| 本质里的词 | 你**必须**要求的图（`figures[].kind`） |
|---|---|
| "**非对称转换**" / 势能驱动 | `potential-landscape` —— 那个"非对称"长什么样？势阱在哪？会卡在哪？ |
| "**非线性**" | `phase-portrait` + `spectrum`（离散峰 vs 连续宽带） |
| "**多体耦合**" | `mode-shape`（简正模） |
| "**混沌**" / "不规律" | `lyapunov-map` + `poincare` + `bifurcation` |
| "**驱动与耗散的平衡**" | 力平衡 + 弛豫时间 + 标度律 |
| 题面里有 **"under certain conditions"** | **`regime-map`（参数空间相图）** ← **这是那条任务的答案** |
| "**多稳态**" | `basin`（吸引域） |

**扫参数画曲线，只能回答"依赖关系"型的任务。"本质"型、"条件边界"型、"模式分类"型的任务，需要的是上面那些图。**

**★ 定完静态图，再问一句：本质是否值得一张 `kind: animation` / `interactive` 图？**
过程本质上是**时空演化**（涡流在壁上流、磁体穿过线圈、模式随参数切换），一张会动的图
比六张静态图更快让人「看懂这个系统是什么」——**但它必须由契约点名，Skill 2 才会做**
（magnetic-brake 的 F-5 涡流动画是契约点名的；electrical-damping 的契约没写，Skill 2
就只能事后补——那个决定本该在这里做）。判据：**模式分类型任务（因变量第三层）几乎总值得**；
纯依赖关系型通常不值得。要求与静态图相同：`expected_shape` 必须预注册。

### ★ `model_validation_checks[]`：验证「中间量」，不只验证最终结果

> **"最终结果对了"不代表"模型对了"——两个错误可以互相抵消，而你永远不会知道。**

**真实做法**（磁铁加速器的报告）：

> *"在中心轴线上取若干点，使用**高斯计**测量磁感应强度，**反推印证磁化强度**"*
> *"**阵列磁铁磁场验证** —— 利用测量的磁化强度计算阵列磁铁磁场"*

**他们没有直接去测末速度然后宣布"模型对了"。他们去测了模型链条中间的那个量（$B$ 场）。**

因为末速度是**一长串环节的乘积**：磁化强度 → $B$ 场 → 力 → 势能 → 运动方程 → 末速度。
**任何一环错了，末速度都错；但末速度对了，不等于每一环都对。**

**挑那些「可以独立验证」的中间量**，每条给**至少两条互不依赖的验证路径**：

| 中间量 | 独立验证的路 |
|---|---|
| **场分布** $B(\mathbf r)$ | ① 轴上闭式解；② 远场极限（必须退化成偶极子）；③ 对称性；④ **高斯计实测** |
| **势能景观** $U(x)$ | ① 极值点 = 平衡点（可直接观察）；② $-\mathrm dU/\mathrm dx$ = 力曲线，**可用拉力计独立测** |
| **力曲线** $F(d)$ | ① 直接测；② 与势能的导数对拍；③ 远场幂次 |
| **本征频率** | ① 小振幅解析解；② 实验频谱峰位 |
| **守恒量** | 关掉耗散，$E$ 必须守恒到数值精度 |

**和 `risky_assumption_checks` 是两回事**：

| | 验什么 |
|---|---|
| `risky_assumption_checks` | **这个简化站不站得住**（物理） |
| `model_validation_checks` | **这个中间量算对了没有**（实现 + 物理链条） |

### ★ `pass_criterion` 的门槛不能比你自己的估计还严

**这是一条机械可判的自相矛盾，而它真的发生过。**

> **血泪教训（magnetic-brake，真实发生）**：A-2（薄壁）的 `criterion_check` 写着"一阶修正 $O(w/a)\approx \mathbf{17\%}$"，而它的 `pass_criterion` 把通过门槛设在 **< 15%**。
>
> **按作者自己的估计，这条假设注定通不过。** 而这个矛盾在写下的那一刻就存在了——不需要跑任何数值就能看出来。（实测 23.4%，比两个数都大。）

**规矩**：如果 `criterion_check` 里给出了一个误差估计 $E$，而 `pass_criterion` 的门槛比 $E$ 还严 —— **那不是"严格"，那是自相矛盾**。要么你的估计错了（那就重估），要么门槛该放宽（那就说清为什么这个精度够用）。

`check_analysis.py` 会机械抓这一条。

### ★ 每条 `RISKY` 假设必须给出**退化特征**（`degenerate_signature`）

**这是 Skill 2 唯一能识破"代码偷懒"的手段，而它必须由你提供——因为只有你知道正确的模型和退化的模型在结构上差在哪。**

> **血泪教训（真实发生）**：F-2 的 `expected_shape` 里埋了个陷阱——"若 Model-2 也给出斜率 4.00，说明有限尺寸修正没被正确实现"。听起来很严密。
>
> 但 Skill 2 的冒烟测试注入了「**点偶极子场 + 正确的径向积分**」（**只少一个修正**），斜率落在 **3.79** —— **既不是陷阱值 4.00，也不是真值 3.44。它从两条断言之间溜过去了。**
>
> **根因**：陷阱是按"整个完整模型偷偷变成玩具模型"校准的。而**真实的 bug 往往只少一个修正**，落点就在两者之间。**一个拟合出来的指数是连续的，可以落到任何地方。**

**所以每条 `RISKY` 的验证任务必须额外给一个 `degenerate_signature`**：

> **如果代码根本没实现这条修正（偷偷退化回玩具模型），会出现什么「结构性」特征？**

**必须是一个在"正确模型"和"退化模型"下取「离散地不同」值的量**：

| 好的退化特征 | 为什么好 |
|---|---|
| **峰的位置** | 点偶极子的涡流峰**解析地**落在 $a/2$；有限长磁体的落在 $L/2$。两个答案，没有中间地带。 |
| **节点 / 零点的个数** | 结构性的，不会连续漂移 |
| **对称性**（严格反对称？） | 要么成立到机器精度，要么不成立 |
| **渐近幂次**（不是拟合值，是**解析可证**的幂次） | |
| **临界点的位置** | |

| 坏的退化特征 | 为什么坏 |
|---|---|
| ~~拟合出来的斜率~~ | **连续量。少一个修正 → 落在陷阱和真值之间 → 溜走。** |
| ~~某个数值的大小~~ | 同上 |

**怎么找**：问自己 —— **"如果我把这个修正项整个删掉，哪个量会*跳变*（而不是平滑漂移）？"那个量就是它。**

## ★★ Stage 8.5 · 判据的双向验证（**交接前的最后一道门，不可跳过**）

> ### **一条判据，只在正确模型上跑过 —— 那不叫验证，那叫「换了一把新的失明的锁」。**

**这一节是被一次真实的、连环的翻车逼出来的**（`examples/electrical-damping/`，两轮审稿）：

1. **r1** 有一条判据在极值点**结构性失明**（一阶判据 $\lvert G'(z_0)\rvert A/\lvert G(z_0)\rvert \ll 1$
   被用在 $\lvert G\rvert$ 的**峰**上 ⟹ 分子恒为零 ⟹ **对任何振幅都报「线性 ✓」**）。审稿抓到了。
2. **r2** 把那把锁拆了，换上三把「更强」的新锁，还报了 $R^2 = 1.000000$、斜率偏差 $+0.0\%$。
3. **r2 的审稿证明：三把新锁全是瞎的。**
   - 「顶点值 $=\gamma_{\rm oc}$」：**完美对中的正确模型偏 $+17.5\%$ ⟹ 判死**（误差棒的 8.8 倍）。
   - 「$R^2=1.000000$」：**75% 的拟合点落在「电磁阻尼已自己关掉」的尾巴上**
     ⟹ **把要测的那个系数改错 30%，三条判据全过。**
   - 「常数阻尼 ⟹ 截距 $\equiv0$」：**捏造的稻草人** —— 两个模型的截距其实**完全相同**。

**为什么三把锁一起瞎？因为只跑了一个方向。**
验证了「新判据在**正确模型**上通过」（$R^2 = 1.000000$，看着真漂亮），
**从没验证「它在**错的模型**上会不会照样通过」。**

### 做法：一张表，两个方向

**写一个 `01-criteria/criterion_matrix.py`，跑出 `01-criteria/matrix.json`。**

| | ★ 正确模型 | 错模型 1 | 错模型 2 | … |
|---|---|---|---|---|
| **判据 A** | ✓ **必须 PASS** | ✗ 抓到 | ✓ | … |
| **判据 B** | ✓ **必须 PASS** | ✓ | ✗ 抓到 | … |

**三个方向都要过**：

| | 查什么 | 违反了叫什么 |
|---|---|---|
| **←** | **正确模型这一列全 PASS** | `CRIT-FALSEKILL` —— **它会误杀一个正确的模型** |
| **↑** | **每个错模型至少被一条判据抓到** | `CRIT-MODEL-UNCAUGHT` —— 它会一路走到评委面前 |
| **→** | **每条判据至少抓到一个错模型** | `CRIT-BLIND` —— **它是一把失明的锁，删掉它** |

### ★ 错模型必须是「学生真的会写出来的东西」——不是稻草人

`wrong_models[].why_a_student_writes_it` 这一栏是**强制的**，而且它是**防稻草人的唯一手段**：

> **说不出「为什么有人会这么写」，那它就是个稻草人 —— 抓到它不证明任何事。**

**至少 3 个。** 好的错模型长这样（都出自真实的两道题）：

| 类型 | 例子 |
|---|---|
| **本题的头号直觉陷阱** | 「磁通最大处阻尼最大」（把 $\lambda$ 当成了 $\mathrm d\lambda/\mathrm dz$）；「用了 $\Phi$ 而不是 $\mathrm d\Phi/\mathrm dz$」 |
| **题面自己诱导的** | "the damping" —— 说「the」就预设了阻尼是**一个数** ⟹ 常数阻尼模型 |
| **最容易犯的实现 bug** | 某个系数错 30%；一个不平凡的积分前因子做错（$45/1024 \to 1/16$）；指数错 1 |
| **★ 最难抓的那个** | **形状全对，只多了一个常数偏置**（线圈不对称 / 磁体蹭壁 / 骨架涡流）。<br>**它的所有标度律一字不差，只有绝对值偏了。**<br>**⇒ 只有「绝对值」或「顶点值」那条判据看得见它 —— 而没有它，那条判据会显得多余。** |

### ★ 拟合区间也是「判据」的一部分，它必须是**算**出来的

r2 的抛物线拟合用了 $\pm6$ mm —— **随手画的**。而在那里 $G$ 的非线性度是 1.1%，
$\Gamma\propto G^2$ ⟹ 顶点偏 $+16\%$ ⟹ **判死正确模型**。
$\pm4$ mm（非线性 $<1\%$）就干净了。

> **区间由物理定。随手画一个范围，就把一条正确的预言变成了一把杀人的锁。**

### ★★★ 而这张表本身是一份**自评** —— 四道 CRIT 门查的全是**布尔值**

**r3 交了一张全绿的表（`verdict: PASS`）。r3 的审稿证明：那张表可以被调到全绿。**

**四道 CRIT 门问的是**：正确模型 PASS 吗？每个错模型被抓了吗？每条判据抓到东西了吗？
**——全是是/否。它们看不见「抓得有多勉强」，也看不见「容差从哪来」。**

**⟹ 写 `criterion_matrix.py` 时，这五条必须自己先过一遍**（否则审稿人会拿 **P18** 打你）：

| # | 必须做 | r3 真实翻的车 |
|---|---|---|
| **①** | **容差必须有来源，而且必须写进契约** | 源码里 `< 0.12`、`< 0.20`、`< 0.15` —— **裸数字**，正文里要么没有、要么写的是 15%（**矛盾**）。<br>而 §9 说 $\Gamma$ 测到 **2%** ⟹ **一条比误差棒宽 6 倍的判据，抓不到一个 10% 的偏置。** |

| **③** | **「★ 正确模型」必须是契约命令 Skill 2 去积的**那个**模型** | 表用 `G_true` = 式 (6) = **Model-0**（点偶极子+薄线圈），<br>而契约的 (26)+(27) 命令 Skill 2 积 **Model-2** ⟹ $c_2$ 差 **+4.1%**<br>⟹ **P1c 的 12% 容差，在 Skill 2 开跑之前就被吃掉 1/3。** |
| **④** | **「不误杀」必须在协议**自己承认的每一项系统误差**上各跑一遍** | `crit_P3` 硬编码 `z0 = 0.0`（**精确的零**），<br>而同一份分析的 §9 写着「**★ 不要试图把磁体对准中心**」。<br>实算：**残余偏心 0.17 mm ⟹ 判死正确模型**，而噪声预算是 ±0.1 mm。<br>**⟹ 判据的有效窗口，必须是一个写下来的数。**（r1 死于 ±0.5 mm，r2 死于 ±6 mm，**r3 第三次**。） |
| **⑤** | **「被抓到」必须是判据的**字面逻辑**抓到的** | naive-A 在三条判据上的「✗ 抓到」，明细全是「**提不出 Γ**」<br>—— 那是 `fit_bernoulli` 在 `len(tp)<6` 返回 `inf`。<br>**⟹ 一个记账截断，被记了五次。`catches` 虚高。** |

> **★ 一个警惕信号：某条判据一口气抓到了**所有**错模型，而且在正确模型上报 $+0.0\%$。**
> **「太好了」本身就可疑。去问：这个拟合的杠杆落在哪一段数据上？**

## ★★★ Stage 8.6 · 契约必须能**自己复算自己**（r3 审稿逼出来的三道门）

> ### **「忘了改」的定义，就是「值没变」。**
> ### **而 `STALE-VALUE` 那道门，第一行就是 `if 值没变: continue`。**

**真实事故**（`examples/electrical-damping`）：
`targets[\gamma].baseline_value = 2.5978` 在 **r1 / r2 / r3 三代一字未动**，
而它自己的公式算出来是 **2.3454**（偏 +10.8%）。
**r2 审稿逐字点过它。r3 的修订记录写着「全部清掉」。而两道 STALE-* 门都没响** ——
因为 `old == new`，门根本没往下走。

> **⟹ 这是 P17 ④ 升了一层楼：这次瞎掉的不是物理判据，是「用来查判据的那道机械门」。**
> **拆掉一把失明的锁之后，你换上的新锁多半也是瞎的 —— 机械门也不例外。**

### 三道门（`check_analysis.py`，已带 `--selftest`）

| 门 | 查什么 | 它抓到了什么 |
|---|---|---|
| **`SPEC-SELFCONTRADICT`** | 每个 `targets[].baseline_value` 必须能由它自己的 **`closed_form`**（可执行 Python）复算出来。**容差按字面量的有效位数定标**：写 s 位 ⟹ 容差 10^(1−s)（clamp 到 [5e-13, 1e-2]）——**baseline 写几位，门就管几位**。固定 1% 的旧门槛抓不到「一位数字的笔误」（实测：`targets[c_2]` 第 4 位 4→5、偏 +0.29%，三代未响） | `targets[\gamma]` = 2.5978 vs 公式的 2.3454；`targets[c_2]` = 0.0345832 vs 0.0344832 |
| **`TARGET-NO-RECIPE`** | 没有 `closed_form` 的 target，**必须**写 `numerical_recipe` 显式声明它无闭式 | **不许沉默地跳过** —— 一个没有任何机械校验的数，可以三代原封不动地错下去 |
| **`CLOSED-FORM-ORPHAN`** | `equations[].closed_form` 里的每个自由标识符，必须在 `parameters`/`symbols`/`targets` 里存在 | `gamma0` —— 一个 r2 就改名成 `\gamma_{oc}`、**在契约里根本不存在**的标识符，三代存活（`STALE-SYMBOL` 只查**正文**，不查 `closed_form` 里的**代码**） |
| **`REVIEW-CITE-GHOST`** | 凡是把一个数归给「r{n} 审稿」的，那个数必须能在 `01-review-r{n}.md` 里**逐字 grep 到** | 一个凭记忆填的 `+27.7%`（r2 报告里出现 **0 次**，它写的是 17.5%）—— **而它已经进了 `CLAUDE.md` 的教训表和审稿人的弹药库** |

### ★ 顺带的收获：`baseline_conditions` 是**必须写下来**的

写 `closed_form` 时会发现一件事：**同一份契约里，不同的 baseline 是在不同条件下算的。**
`\gamma` 和 `c_2` 用 **R = 20 Ω**；`t^*` 和 `A_c` 用**短路 R = 0**。
**这个区别以前只活在作者脑子里。**

> **⟹ 一个 baseline 复算不出来，第一嫌疑不是「值错了」，而是「它的基准条件从来没被写下来」。**

## Stage 9 · 被 Skill 2 的数值结果挑战时，怎么修订

Skill 2 算完，可能带着一份 `02-sim/model-challenge-r{n}.md` 回来找你：**你承诺的曲线形状，数值算出来不是那样。**

它已经先自证清白了——`model-challenge` 里必须附 **Gate 0（极限对拍）的通过记录**，那是纯数学恒等式，证明"不是我代码错"。所以问题在物理这边。

**这一刻是整个流水线上最危险的一刻。** 因为"改模型让它对上数值"比"承认模型错了"舒服得多，而且**改完读起来完全合理**。这是审稿模式 **P16（事后合理化 / HARKing）**，而 Physics Fight 上 Opponent 的杀招只有一句话：

> **"你这条预测，是在算之前写的，还是算之后写的？"**

### 修订的四条铁律

1. **修订必须指向一个物理理由。** 漏了一项 / 符号错 / 边界条件错 / 某条假设崩得比预想更厉害。
   > **"为了和数值吻合"不是理由。**
2. **单向棘轮：只能让预测变弱。** 加适用条件、缩小适用域、把某条预测降级。**不能凭空变强。**
   > 修订之后预测反而变得更精确、更漂亮——**高度可疑**，那多半是照着答案描出来的。
3. **新的 `expected_shape` 必须能从修订后的模型独立推导出来**，推导要写进 `01-analysis.md`。
   > **若新的 `expected_shape` 恰好就等于数值结果，那它不是预测，是事后拟合。**
4. **必须留痕。** 修订前的 `handoff/model-spec.json` 归档成 `model-spec-r{n}.json`（**不覆盖**）。这条归档链是时间戳化的物证——审稿人会逐版 diff 它。

### 修订完必须重新过审

重跑 `check_analysis.py`，重派审稿人，并在审稿 prompt 里**明确写上**：

> **"这份分析是被数值结果逼着修订的。请重点查 P16（事后合理化）：拉出 `model-spec-r{n}.json` 的归档链逐版 diff，问每一处改动——这是物理理由，还是只是为了和数值对上？"**

### 但先分清楚：这真的是"被打脸"吗

**如果 `risky_assumption_checks[].pass_criterion` 里事先就写了"若 X 则做 Y"，而现在 X 成立了——那是预注册（pre-registration），不是被打脸。**

真实例子（magnetic-brake 的 A-1）：

> *"若 $|k-4| > 0.3$，则 P5 被数值判定为不可靠，**必须在 `01-analysis.md` 的预测表中把 P5 降级并说明**。预期 k 明显小于 4——**这不是失败，是模型边界被正确定位**。"*

实测 $k = 3.44$。触发了，**照办即可**——`status: PRESCRIBED-REVISION`，**不走审稿循环**。

**执行一条在数据出现之前就写好的应对动作，与"看到数据之后编一个应对"，是完全不同的两件事。** 前者是临床试验的金标准，后者是 P16。

> **所以：在 Stage 7 写 `pass_criterion` 时，尽量把"如果不通过该怎么办"也预先写死。** 这不只是严谨——它是你事后免于被指控 HARKing 的**唯一**护身符。

**回环最多 2 轮**（见 `docs/pipeline.md` §5.3）。超限 → 文首 `[GAP]` + `open_gaps[]`，诚实弃权。

## 完成的标准

不是"写得长"，是这八件事都做到了：

1. **★ 任务是挖出来的**，每条都带 `excavated_from`（题面里的哪个词）；题面里**没有任何限定词被漏掉**
2. **★ 因变量列到了第三层「模式」**，不只是标量和函数
3. **★ 说得出一句话的物理本质**，而且它**决定了图的骨架**（不是事后补的修辞）
4. 每条简化都被显式记账，且带**不等式**判据（不是"通常成立"）；**判据列全了所有相关尺度**
5. 每个忽略都有**数字**，且做过**扫描端点检查**
6. 有一个**零自由参数的预测**（幂律指数 / 临界值 / 坍缩曲线）
7. 模型**分层**：Model-0 能手算能看懂，Model-2 才上数值
8. **说得出"如果 XX 假设不成立，我的结论会变成 YY"**

## 常见的自我背叛

- **★ 把题面直接当任务书。** "Investigate the phenomenon" 不是任务，它是**三条**任务的种子。**题面里每一个限定词都要被追问一次**——"under certain conditions" 里藏着一整张参数空间相图。
- **★ 因变量只列了"数"，没列"模式"。** 于是你交了一堆 $x(t)$ 曲线，而题目问的是"有哪几种运动模式、边界在哪"。**这是 IYPT 的分水岭。**
- **★ 物理本质写成"研究磁铁在管中的运动"。** 那是**题目**，不是**本质**。本质是"驱动与耗散的平衡"——它告诉你该画力平衡图。**说不出本质 = 不知道该画什么图。**
- **★ 图是"扫几个参数画几条曲线"凑出来的**，不是从本质推出来的。于是"本质"型和"条件边界"型的任务一条都没答上。
- **★ 只打算用"最终结果吻合"来证明模型对。** 两个错误可以抵消。**去验证中间量**（$B$ 场、势能、力曲线）——**IYPT 的报告是拿高斯计去测 $B$ 场的，不是拿末速度去反证模型。**
- **写了假设台账，但判据是"通常成立"** —— 等于没写。判据的价值在于它能代入数字然后打你的脸。
- **判据只写了一个尺度**（"$a \gg L$"）—— 漏掉的那个（$R$）会让极限对拍取错极限。**远场判据必须列全源的每一个尺度。**
- **忽略了空气阻力，理由是"空气阻力很小"** —— 给数字。
- **在基准参数下算了忽略的比值，但图扫了三个数量级** —— 扫描端点检查。
- **用了小角近似，然后画了一张 θ 从 0 扫到 90° 的图** —— 这是同时犯 P7 和 P3，常见得可怕。
- **审稿人给了 PASS，但它没真的读 failure patterns** —— 它在同意你，不是在检验你。看它的报告里有没有那张"逐条结果"的表。
