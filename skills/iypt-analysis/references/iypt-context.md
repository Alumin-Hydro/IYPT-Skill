# IYPT 是什么，以及它如何决定"充分的分析"长什么样

## 赛制：Physics Fight

IYPT 不是做题考试，是**辩论**。一场 Physics Fight 里有三个角色：

| 角色 | 干什么 |
|---|---|
| **Reporter（正方）** | 报告自己对这道题的研究：物理机制、理论模型、实验、理论与实验的对比 |
| **Opponent（反方）** | **攻击**这份报告：找出假设的漏洞、忽略的机制、实验的系统误差、理论与实验对不上的地方 |
| **Reviewer（评论方）** | 评价双方的表现 |

**关键推论：分析必须为"被攻击"而写，不是为"被阅读"而写。**

Opponent 不需要证明你错。他只需要问出一个你答不上来的问题：
- "你说空气阻力可忽略，忽略到什么程度？给个数。"
- "你用了点偶极子近似，但管壁离磁体只有 3 mm，而磁体长 10 mm，$r \gg L$ 成立吗？"
- "你的图从 0 扫到 90°，但你的模型用了小角近似。"
- "你的模型有 3 个拟合参数，数据只有 2 个特征——'吻合很好'能说明什么？"

**这四个问题，正是 `iypt-physics-review` 那 15 条错误模式的来源。** 分析里每一条被显式记账、带判据、带数字的假设，都是提前拆掉的一颗子弹。

## 题目为什么是欠定的

IYPT 的题干长这样：

> "A cylindrical magnet is dropped into a vertical conducting pipe. Investigate the motion of the magnet and how the terminal velocity depends on relevant parameters."

它**故意**不告诉你：磁体多大、管子什么材料、壁厚多少、"relevant parameters" 是哪些、扫描到什么范围。

**这不是题目没写完，这是题目的一部分。** 做出这些选择、并说明为什么这么选——**本身就是得分点**。一个把参数范围选得能同时展示"黏性主导"和"惯性主导"两个区间的人，比一个随便选了一组参数的人，分数高得多。

**这就是为什么 Skill 1 的 Stage 1 是「设定书」而不是直接开始建模。** 补全题目不是在填空，是在设计一个实验。

## 评分维度（决定了 `01-analysis.md` 里该有什么）

| 维度 | 分析里对应什么 |
|---|---|
| **物理机制的理解** | Stage 4 机制预算：说得出哪些机制在起作用、各自多大、谁主导 |
| **理论模型的深度与自洽** | Stage 5 分层推导 + Stage 6 自洽性检验 |
| **假设的合理性与自觉** | Stage 2 假设台账。**"我知道我什么时候会错"是深度的标志，不是弱点** |
| **实验设计与数据质量** | 实验方案（见下） |
| **理论与实验的定量对比** | Stage 7 可证伪预测——**必须有不含自由参数的预测**（幂律指数、无量纲组临界值、坍缩曲线），否则"吻合"没有证据价值 |
| **对参数依赖的探索** | 参数扫描范围（设定书） + `figures[]` |
| **表述的清晰** | Skill 3 |

**分数最集中的地方不是"模型多复杂"，是"你多清楚自己模型的边界在哪"。** 一个简单但知道自己何时失效的模型，胜过一个复杂但不知道自己何时失效的模型。

## 关于实验：一个必须诚实的说明

**真实 IYPT 要求真正动手做实验，且实验占很大权重。** 这套 skill 流水线用**数值仿真**替代实验——这不是等价物。仿真验证的是"我的方程解对了"，实验验证的是"我的方程写对了"。**只有实验能证伪模型。**

因此：

1. `01-analysis.md` **必须包含一个可执行的实验方案**（器材、测量方法、精度要求、误差来源、如何提取要对比的量）。这是给人去做的，不是走过场。
2. 仿真结果在任何场合都**必须明确标注为仿真**，不得伪装成实验数据。这是底线，越过了就不是物理了。
3. 分析里要说清：哪些结论已被仿真支持，哪些**仍然需要真实实验才能确认**。

## 题型分类（Stage 0 用）

| 题型 | 题干特征 | "充分分析"的重心 |
|---|---|---|
| **现象解释型** | "Explain the phenomenon" / "Why does X happen" | 机制预算是核心——先证明**是这个机制**，排除其他候选机制。必须解释清楚是什么导致了现象 |
| **依赖关系探究型** | "Investigate how Y depends on relevant parameters" | 量纲分析 + 参数扫描是核心。**最常见的题型**。要给出幂律指数、坍缩曲线、区间划分（哪个区间哪个机制主导） |
| **装置优化型** | "Maximize / optimize X" | 建立目标函数与约束，找出**权衡关系（trade-off）**——什么增大了、什么必然减小了。最优点通常出现在两个机制交界处 |
| **测量方法型** | "Determine / measure X using Y" | 误差分析是核心：灵敏度 $\partial X/\partial(\text{观测量})$、系统误差、精度极限 |

题型判错，重心就放错了。**"解释为什么"的题，你交了一堆参数扫描图，是跑题。**
