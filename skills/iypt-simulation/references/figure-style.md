# 出图风格：给 Physics Fight 用的图

图不是插图，是**论据**。这份文档管的是"论据能不能被看清、能不能被验证"，不是"好不好看"。

**产出环境有三个硬约束，它们决定了一切：**

1. **投影到会议厅大屏**，后排要能读 → 大字号、高对比
2. **可能被打印成灰度**（评委手里的讲义）→ **色相会完全消失**
3. **观众里有色盲**（男性约 8%）→ 色相会部分消失

**结论：身份永远不能只靠颜色。** 色 + 线型 + 标记，三重冗余。

---

## 1. 语义色彩：Model-0 不是"系列 2"

几乎每张 IYPT 图的骨架都是「解析预测 vs 数值结果」。**这两者的地位不对等**：

- **Model-0（闭式解 / 解析预测）= 被检验的假说**，是零假设，是**参照**
- **Model-2（数值解）= 检验的结果**，是**发现**

把它们画成两个平等的 categorical series，是**语义错误**。科学制图的惯例（也是正确的）：

| 角色 | 颜色 | 线型 | 标记 |
|---|---|---|---|
| **Model-0 · 解析参照** | 中性墨 `#52514e`（7.94:1） | **虚线** `--` | **无** |
| **Model-1 · 修正** | slot 2 green `#008300` | 点划 `-.` | `s` 方 |
| **Model-2 · 数值结果** | slot 1 blue `#2a78d6`（4.42:1） | **实线** `-` | **`o` 圆** |

参照线是**退让的**（recessive），结果是**主导的**（dominant）。读者一眼就该知道哪条是"我算出来的"，哪条是"要被打脸的那条"。

## 2. 标记 = 真正算过的点（这是诚实性规则，不是美学规则）

- **解析解可以在任意点求值** → 画平滑曲线，**不画标记**
- **数值解只在有限个点上算过** → **画标记**，标记就落在你真算过的地方

> 一条光滑无标记的"数值解"曲线，是在对读者撒一个小谎——它假装你有无穷多个点。
>
> **Opponent 会问："这条曲线你算了几个点？"** 图上有标记，答案就在图上。

## 3. 分类色板（已验证，不许眼睛调）

**投影优先序**——五个槽全部清过白底 3:1：

| 槽 | 色 | 对比度 | 线型 | 标记 |
|---|---|---|---|---|
| 1 | blue `#2a78d6` | 4.42:1 | `-` 实线 | `o` 圆 |
| 2 | green `#008300` | 4.95:1 | `-.` 点划 | `s` 方 |
| 3 | violet `#4a3aa7` | 8.56:1 | `:` 点线 | `^` 上三角 |
| 4 | red `#e34948` | 3.95:1 | 长划 `(0,(6,2))` | `D` 菱形 |
| 5 | orange `#eb6834` | 3.20:1 | 划点点 `(0,(4,1,1,1,1,1))` | `v` 下三角 |

**槽位按顺序分配，绝不循环。** 第 6 个系列不是"再生成一个色"——它意味着你该拆成小多图（small multiples）了。

### 验证记录（规则要求：WARN 必须有对应的 relief 通道）

```
node scripts/validate_palette.js "#2a78d6,#008300,#4a3aa7,#e34948,#eb6834" \
     --mode light --surface "#ffffff" --pairs all
```

- ✅ **Lightness band** — 5/5 落在 L 0.43–0.77
- ✅ **Chroma floor** — 5/5 ≥ 0.10
- ✅ **Contrast vs surface** — 5/5 **≥ 3:1**（3.20 – 8.56）
- ⚠️ **CVD separation** — 全配对最差 **ΔE 11.2**（orange↔green，protan），落在 8–12 的 floor 带

**这条 WARN 是合法的**，因为规则写明"floor band is legal **ONLY** with secondary encoding"——**而我们的冗余编码是强制的**（灰度打印本来就要求它）。色盲读者根本不靠色相区分这两条线，靠的是 `v` vs `s` 和"划点点" vs "点划"。

> **为什么不选 CVD 更好的那组**：把 orange 换成 aqua `#1baf7a`，CVD 能升到 13.3，但 aqua 对白底只有 **2.82:1**。**投影的有效对比度比显示器更差**——2.82 的线打到会议厅后排基本看不见，而"加直标"救不回来（标签看得见，但你没法用眼睛去追那条曲线）。
>
> **拿 WARN 换 WARN 时，选那个能被你已有的冗余通道救回来的。** 对比度救不回来，CVD 能。

### 用色板的三条铁律（来自 dataviz 方法，普适）

- **颜色跟着实体走，不跟着排名走。** 筛掉一个系列，剩下的**不许重新上色**。
- **绝不用双 y 轴。** 两个量纲不同的量 → 两张图，或小多图，或归一化到同一基准。（这是制图的头号错误。）
- **绝不用彩虹色阶做 sequential。** 单色相，浅→深。

## 4. 投影排版

| 项 | 值 | 为什么 |
|---|---|---|
| 轴标签 | ≥ 14 pt | 后排要读 |
| 刻度 | ≥ 12 pt | |
| 标题 | ≥ 16 pt | |
| 线宽 | **2.0** | matplotlib 默认 1.5 太细 |
| 标记 | ≥ 8 px | |
| 网格 | `#e1e0d9`（1.32:1） | **退让**——网格是参考，不是内容 |
| 轴脊 | 只留左、下 | 上、右两条是噪声 |
| 图例 | **直标优先** | 图例框逼观众来回扫；直标就在线旁边 |

**轴标签必须带单位**，且与 `symbols[].unit` 对齐：`v_t (m/s)`，不是 `v_t`。

## 5. `SIMULATION` 戳 —— 强制，无例外

**每张图必须盖。** 内容不只是 "SIMULATION"，还要写清是哪个模型、哪一版契约：

```
SIMULATION · Model-2 · spec r1
```

**机械校验**：`figkit.py` 同时存 `.png` 和 `.svg`。SVG 是纯文本 XML，`check_sim.py` 直接在里面 grep `SIMULATION` 字样。**这个检查伪造不了——除非你真的盖了戳。**

> **为什么这是底线而不是排版偏好**：真实 IYPT 的评分里实验占很大权重。一张没标注的仿真图放进 PPT，在 Physics Fight 上会被当作**伪造实验数据**。这是学术不端，不是疏忽。
>
> 仿真验证"方程解对了"，实验验证"方程写对了"。**只有实验能证伪模型。**

## 6. 断言结果画在图上

**读者不该需要翻 `results.json` 才知道这张图过没过。**

loglog 图：**把拟合斜率标在线上**（别让观众拿尺子量），并把 expected 写在旁边：

```
┌─────────────────────────────────┐
│  Model-0:  k = 4.00  (预期 4.00) ✓ │
│  Model-2:  k = 3.44  (预期 ≪ 4)  ✓ │
│  拟合区间: a ∈ [5.5, 12] mm       │
└─────────────────────────────────┘
```

**拟合区间必须写出来。** 换个区间斜率会变——因为它本来就不是严格的幂律。不写区间的斜率是不可复现的数字。

## 7. 数据坍缩图（F-4 那一张）

**这是杀伤力最大的一张图，给它版面。**

它的信息不是"每个点是谁"，是"**它们重不重合**"：

- **Model-0 的点**：全部中性墨 + 同一个标记 → 它们会**堆到同一条水平线上**。这个"堆叠"本身就是信息。
- **Model-2 的点**：5 个 categorical 槽 + 5 种标记 → 它们**散开**，而且你能看出每个都是一次真正独立的运行。
- **理论水平线**（如 $\Pi_1 = 1024/45 = 22.756$）：细虚线 + 直标。

**坍缩失败不是画图失败。** magnetic-brake 的 Model-2 散布 44.9%——这**正是结论**（A-1 崩溃引入了第三个无量纲组 $L/a$）。不许为了"图好看"去挑参数组合让它坍缩。

## 8. 交互页面（`kind: animation` / `interactive`）

- **自包含单文件 HTML**：无 CDN，无外部字体，无 fetch。可以直接发给队友、塞进 PPT、离线放。
- **数据从 Python 结果烘焙进去**（内联 JSON）。**页面不许自己算物理**——否则它会算出一套和 `results.json` 不一致的数，而没有任何检查能发现。
- 明暗两套主题（`prefers-color-scheme`）——投影环境可能是任意一种。
- 同样要盖 `SIMULATION` 戳。

## 9. 反模式清单

| 反模式 | 为什么是错的 |
|---|---|
| **曲线与 `expected_shape` 不符，于是把图改好看** | **头号罪。** 图是用来证伪的，不是用来配色的。 |
| 双 y 轴 | 两个尺度叠在一起，任何"相关性"都是画出来的 |
| 光滑无标记的数值曲线 | 假装你有无穷多个点 |
| 幂律斜率不标拟合区间 | 不可复现 |
| 3D 柱 / 饼图 / 阴影 | 遮挡数据，制造虚假体积感 |
| 每个点都标数字 | 全标 = 全不标。选择性直标。 |
| 仿真图不盖 `SIMULATION` | 学术不端 |
| 用状态色（红=坏）当"系列 4" | 状态色有保留含义，不许冒充系列 |
| 第 6、7、8 个系列还在生成新色 | 该拆小多图了 |
