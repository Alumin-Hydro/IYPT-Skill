# 数值实操：会真的咬人的坑

不是数值分析教程。只收录**在 IYPT 这类题里真的会让你算出错误答案、而且看不出来**的东西。

---

## 1. 先找解析化简，再上数值

**数值积分套数值微分，是噪声的放大器。** 动手写代码之前，花十分钟找化简——常常能把一层积分直接消掉。

### 实例：Leibniz 求导消掉一整层积分

magnetic-brake 的 (15) 需要 $\partial\Phi/\partial z$，其中 $\Phi(r,z)$ 是有限长均匀磁化圆柱在半径 $r$ 圆环上的磁通。

安培模型（侧面束缚面电流 $K = M_s$）把它写成对共轴圆环互感 $M(R,r,d)$ 的积分：

$$\Phi(r,z) = M_s\int_{-L/2}^{L/2} M(R,r,z-z')\,dz' \;\overset{u=z-z'}{=}\; M_s\int_{z-L/2}^{z+L/2} M(R,r,u)\,du$$

**两个积分限都含 $z$，被积函数不含 $z$** → Leibniz 求导，积分整个消失：

$$\boxed{\ \frac{\partial \Phi}{\partial z} = M_s\Big[M(R,r,z+\tfrac{L}{2}) - M(R,r,z-\tfrac{L}{2})\Big]\ }$$

**闭式。** 不需要数值微分，不需要外层积分。原本的三重数值积分变成两次特殊函数求值 + 双重积分。

**副产品是物理洞察**：这个式子就是"磁体两个端面的等效电流环之差"——它天然是 $z$ 的奇函数，于是涡流分布是**反对称双峰**（前方排斥、后方吸引，两者都阻碍下落）。这正是 Lenz 定律的样子。

> **能解析给出的导数，永远不要数值求。** `numerical_notes` 里那句"dΦ/dz 建议解析给出，避免数值微分放大噪声"，说的就是这个。

## 2. 特殊函数：约定和缺失

### `scipy.special.ellipk(m)` 吃的是**参数 $m = k^2$**，不是模数 $k$

经典陷阱。文献里的公式几乎都用模数 $k$ 写，scipy 用参数 $m$。传错了不会报错，只会**默默给你一个错的数**。

```python
k2 = 4*R*r / ((R+r)**2 + d**2)      # 这是 k^2
K = ellipk(k2)                       # ✅ 对：scipy 要 m = k^2
K = ellipk(np.sqrt(k2))              # ❌ 错：但它不会报错
```

**Mathematica 的 `EllipticK[m]` 同样用参数；而很多教科书和 Abramowitz-Stegun 用模数。抄公式前先确认对方用的哪个。**

### scipy **没有**完全第三类椭圆积分 $\Pi(n,k)$

有 `ellipkinc` / `ellipeinc`（不完全第一、二类），有 `ellipk` / `ellipe`（完全第一、二类），**没有完全第三类**。

有限长螺线管的离轴磁场，标准公式里带 $\Pi(n,k)$（或 Heumann's lambda）。**照着硬上，你会撞墙**，然后要么自己实现 Bulirsch 的 `cel` 算法，要么去装第三方库。

**绕路**：走**共轴圆环互感**的路线，只需第一、二类：

$$M(R,r,d) = \mu_0\sqrt{Rr}\left[\left(\tfrac{2}{k}-k\right)K(k) - \tfrac{2}{k}E(k)\right],\qquad k^2=\frac{4Rr}{(R+r)^2+d^2}$$

**换一个等价的物理表述，往往能换掉一个你没有的特殊函数。** 这比自己实现 `cel` 便宜得多，也不容易错。

### $k^2 \to 1$ 时 $K(k)$ 对数发散

$k^2 = 1$ 发生在 $R = r$ 且 $d = 0$——即两个圆环重合。

物理上：**磁体半径逼近管内半径时**。magnetic-brake 基准 $R=5$mm、$a=6$mm → $k^2_{\max} = 4\cdot5\cdot6/11^2 = 0.9917$，$K \approx 3.7$——**大但有限**。

**但如果你把 $a$ 扫到接近 $R$，或者磁体几乎塞满管子，这里会炸。** 加一个断言把它挡住，别让它悄悄污染结果。

## 3. 广义积分 $\int_{-\infty}^{\infty}$：截断是隐形杀手

截短了，结果**系统性偏小**，而且偏得平滑——**曲线看起来完全正常**，只是每个点都错了一点，而且错的量随参数变化。于是你的**斜率**是错的。

三条规矩：

1. **用对称性**。被积函数是偶函数（$\partial_z\Phi$ 是奇的 → 平方是偶的）→ $\int_{-\infty}^{\infty} = 2\int_0^{\infty}$。少一半工作量，也少一半出错机会。
2. **截断长度用自然尺度的倍数，不要用绝对值**。这里是 $z_{\max} = 200a$，不是 `0.2`——**参数扫描时 $a$ 会变，绝对截断长度会在扫描的一端悄悄失效**。
3. **翻倍再算一遍**（Gate 1）。这是唯一能证明截断够长的办法。

被积函数的衰减：$\partial_z\Phi \sim z^{-4}$（偶极子极限），平方后 $\sim z^{-8}$。衰减很快，$200a$ 绰绰有余——**但这是算出来的，不是猜的。**

## 4. 嵌套 `quad`：容差不要贪

嵌套求积（外层 `quad` 的被积函数里又调 `quad`）时，**内层的误差是外层的噪声**。把 `epsrel` 压到 `1e-11` 会触发：

```
IntegrationWarning: The occurrence of roundoff error is detected,
which prevents the requested tolerance from being achieved.
```

**这不是"差一点点"，是 quad 在告诉你它已经在跟浮点噪声搏斗了。**

经验：嵌套两层时，内层 `epsrel=1e-10` 就到头了。**要更高精度，得改算法（换成解析化简、或用高斯求积固定节点），不是拧容差。**

而且——**Gate 0 的误差如果卡在 $10^{-5}$ 量级不再下降，先怀疑是求积精度的地板，而不是模型错**。magnetic-brake 的 Gate 0 在 $\epsilon = 0.003$ 时误差停在 0.0036% 不再改善，就是这个地板。

## 5. 幂律拟合：斜率是零自由参数预测，也是最容易造假的数字

**幂律指数是 IYPT 报告里最有说服力的数字**——它没有自由参数，可以被杀死。正因如此，它也是最容易被无意识地美化的数字。

三条：

1. **先检查它到底直不直。** 在 log-log 上拟合一条直线，**不管数据直不直，你都会得到一个斜率**。拟合前先看残差 / $R^2$；有系统性弯曲的，"斜率"是没有意义的——**弯曲本身才是结论**（比如 F-1 的 Model-2 在大 $w$ 端上翘）。
2. **报告拟合区间。** "斜率 3.44" 是在 $a\in[5.5,12]$ mm 上拟的。换个区间会变——**因为它本来就不是严格的幂律**。区间必须写出来。
3. **端点效应**。扫描范围的端点常常是物理机制切换的地方（正是 Skill 1 精心选的），把它们算进拟合会污染斜率。**要么分段拟合，要么明确说"全域拟合，因此这是个有效指数"。**

Model-0 的斜率应当**精确**等于解析值（4.0000 / −1.0000）。**不精确 → 你的 Model-0 代码或参数错了，先修这个**——它是你唯一的地面真值。

## 6. 刚性与多尺度：时间尺度差 4 个数量级

magnetic-brake：$\tau = M/b = 2.8$ ms，全程 36 s。**差 4 个数量级。**

- 画整段轨迹用线性时间轴 → **瞬态被压成一条竖线，什么都看不见**
- 用对数时间轴 / 对数位移轴（F-3 就是 `semilogx`）
- 有闭式解就用闭式解（$v = v_t(1-e^{-t/\tau})$），数值解只用来对拍

**"看不见"和"不存在"是两回事。** 一条看起来完全水平的 $v(t)$ 曲线，可能藏着一个你没画出来的、决定性的瞬态。

## 7. 参数一律从 `model-spec.json` 载入

**不许硬编码。**

硬编码的数字会和契约悄悄漂移——你改了 spec 里的 $a$，代码里还是老值，而且**没有任何检查能发现**。反向边修订过 model-spec 之后，这一条尤其致命。

```python
# ✅
spec = json.loads(Path("handoff/model-spec.json").read_text(encoding="utf-8"))
P = {p["symbol"]: p["value"] for p in spec["parameters"]}
a = P["a"]

# ❌
a = 6.0e-3   # 管内半径
```

**唯一例外**：物理常数（$\mu_0$、$g$）——但它们通常也在 `parameters[]` 里，那就还是从那里读。

## 8. Windows

- **Python 从 stdin 读中文源码会按 GBK 解码 → 乱码。** 脚本写成文件再跑，不要 `python -c` 或管道喂进去。
- **控制台输出中文要显式 reconfigure**，否则 `UnicodeEncodeError`：

```python
import sys
for s in (sys.stdout, sys.stderr):
    try:
        s.reconfigure(encoding="utf-8")
    except (AttributeError, OSError):
        pass
```

- **没有 ffmpeg** → matplotlib 存不了 mp4。**动画走自包含 HTML 页面**（本来就该这么做：单文件、可直接发给队友、可塞进 PPT）。
