# 本质探究工具箱：仿真不只是"把那个数算出来"

**仿真有三个职责，`tasks[]` 只占其中一个。**

| 职责 | 干什么 | 不做的后果 |
|---|---|---|
| **1. 回答任务** | 把 `tasks[]` 一条条答掉 | 跑题 |
| **2. 验证建模** | **对拍中间量**，证明模型的每一环都对 | **最终结果对了，但两个错误互相抵消——你不知道** |
| **3. 阐释本质** | 回答"这到底是个什么系统" | 你交了一堆曲线，但没人（包括你）知道这个系统**是什么** |

**新手只做 1。IYPT 的分数在 2 和 3。**

# 职责 2 · 验证建模：**对拍中间量，不只对拍最终结果**

> **"最终结果对了"不代表"模型对了"** —— 两个错误可以互相抵消，而你永远不会知道。

### 真实做法（磁铁加速器，IYPT 报告）

> "在中心轴线上取若干点，使用**高斯计**测量磁感应强度，**反推印证磁化强度**"
>
> "**阵列磁铁磁场验证** —— 利用测量的磁化强度计算阵列磁铁磁场（此时底板不是金属板）"

**他们没有直接去测末速度然后宣布"模型对了"。他们去测了模型链条**中间**的那个量（$B$ 场）。**

**因为末速度是一长串环节的乘积**：磁化强度 → $B$ 场 → 力 → 势能 → 运动方程 → 末速度。
**任何一环错了，末速度都错；但末速度对了，不等于每一环都对。**

### 中间量清单 —— 挑那些**可以独立验证**的

| 中间量 | 怎么独立验证（**至少两条互不依赖的路**） |
|---|---|
| **场分布** $B(\mathbf r)$、$E$、$T$、$p$、$\mathbf v$ | ① 轴上/对称轴的**闭式解**；② **远场极限**（必须退化成偶极子/点源）；③ 对称性（奇偶、反对称）；④ **实测**（高斯计 / 热电偶 / PIV） |
| **势能景观** $U(x)$ | ① 极值点位置 = **平衡点**（可以直接观察）；② $-\mathrm{d}U/\mathrm{d}x$ = **力曲线**，可以用拉力计独立测 |
| **力曲线** $F(d)$ | ① 直接测（拉力计 / 弹簧秤）；② 与势能的导数对拍；③ 远场极限（$F \propto d^{-4}$ 等） |
| **本征频率** | ① 小振幅极限下的**解析解**；② 实验频谱的峰位 |
| **守恒量** | 关掉耗散，$E$ 必须**守恒到数值精度**（这是积分器的 Gate 0） |
| **无量纲数** | 代入基准参数，量级对吗？（$Re$、$Re_m$、$We$…） |

> **每一个中间量都是一道独立的门。**
>
> 中间量错了、最终结果却对了 —— **那是巧合，不是物理。而 Opponent 会去查中间量。**

### 契约里的位置

Skill 1 在 `model_validation_checks[]` 里指定要验哪些中间量、怎么验。
**Skill 2 必须全跑，并把结果写进 `results.json` 的 `validation_checks[]`。**

这和 `risky_assumption_checks[]` **是两回事**：

| | 验什么 |
|---|---|
| `risky_assumption_checks` | **这个简化站不站得住**（物理） |
| `model_validation_checks` | **这个中间量算对了没有**（实现 + 物理链条） |

# 职责 3 · 阐释本质

## 第一步：读 `essence.one_sentence`

Skill 1 在契约里写了一句话。**它决定了你要画哪些图。**

| 本质里的词 | 你**必须**画的图 |
|---|---|
| "**非对称转换**" / "势能驱动" | **势能景观 $U(x)$** —— 那个"非对称"长什么样？ |
| "**非线性**" | 相图、**FFT 频谱**（离散峰 vs 宽带） |
| "**多体耦合**" | **简正模**、模式分解 |
| "**混沌**" / "不规律" | **Lyapunov 指数**、Poincaré 截面、**分岔图** |
| "**驱动与耗散的平衡**" | 力平衡图、弛豫时间、**标度律** |
| "**在某些条件下**"（题面有 "under certain conditions"） | **★ 参数空间相图（regime map）** ← **这是那条任务的答案** |
| "**多稳态**" | **吸引域（basin）** |

## 工具逐条

### 1. 相图 (phase portrait)

**画什么**：状态空间 $(x, \dot x)$ 或 $(\theta, \dot\theta)$ 里的轨迹、不动点、极限环、分界线 (separatrix)。

**回答什么**：系统的**定性结构**。周期？准周期？发散？被困在势阱里？

> **陷阱：只画一条轨迹 = 没画相图。** 要画**一族**初始条件——相图的信息在"轨迹之间的关系"里，不在单条轨迹里。

### 2. 势能景观 $U(x)$

**磁铁加速器的核心就是这张图**：

> "磁铁阵列的结构导致磁铁车的**磁势能沿前进方向呈波动中下降趋势**，形成加速"

**图上要能一眼看出**：势阱、势垒、下降趋势、**局部极小**（车会卡在哪）。

**加分**：力 $= -\mathrm{d}U/\mathrm{d}x$ **可以独立测量** → **这张图同时是一道模型验证门**。

### 3. FFT 频谱

**判周期性最便宜的手段。先做它，再做 Lyapunov。**

| 频谱长相 | 结论 |
|---|---|
| **离散锐峰** | 周期 / 准周期 |
| 峰的个数 = 简正模个数 | 线性耦合主导 |
| 出现**组合频率**（$\omega_1 \pm \omega_2$） | 非线性开始起作用 |
| **离散峰展宽为连续宽带** | **混沌** |

> 真实报告的原话：*"离散的简正模频率峰逐渐展宽为连续宽带谱 —— 系统从准周期运动过渡到混沌"*

### 4. 简正模 / 线性化

在平衡点线性化 → 本征值 / 本征矢 → 简正频率 $\omega_k$ 和模式形状。

**★ 这同时是一道验证门**：**数值解在小振幅极限下必须回到简正模的频率。** 不回到 = 代码错。

### 5. Lyapunov 指数 $\lambda$ —— 混沌的**定量**判据

$\lambda > 0 \iff$ 混沌（相邻轨迹**指数**分离）。

**标准算法（Benettin 重正化法）**：

```
1. 取两条初始条件，相距 d0（很小，如 1e-8，但要远大于机器精度）
2. 演化 Δt，测新的距离 d1
3. 累加 ln(d1/d0)
4. ★ 把第二条轨迹沿分离方向拉回到距离 d0（重正化）—— 否则它会跑出线性区
5. 重复 N 次，λ = (1/(N·Δt)) · Σ ln(d1/d0)
```

**★ 必须做的四个检查（否则你算的是噪声）**：

| 检查 | 为什么 |
|---|---|
| **λ 对积分时长收敛吗？** | 短时间的 $\lambda$ 完全是暂态噪声。**画 $\lambda(T)$ vs $T$，看它是否趋于平台。** |
| **无耗散的可积系统 $\lambda$ 必须 ≈ 0** | ← **这是 Lyapunov 代码的 Gate 0。** 拿单摆（小振幅）或谐振子去打，$\lambda$ 必须落回 0（在数值噪声内）。不落回 = 代码错。 |
| **$\lambda$ 对 $d_0$ 和重正化间隔 $\Delta t$ 不敏感吗？** | 收敛门。$d_0$ 太大 → 出线性区；太小 → 淹在浮点噪声里。 |
| **有耗散时 $\sum\lambda_i < 0$** | 相空间体积必须收缩。 |

> **杀伤力最大的产出：$\lambda$ 在参数平面上的二维热图**（如 $\lambda$ vs (间距 $d$, 磁场 $B$)）。
>
> **一张图给出整个混沌相图。** 真实报告里就是这么干的（`FIG.8 λ VS d & B`）。

### 6. 分岔图 (bifurcation diagram)

x 轴 = **参数**；y 轴 = 长时间之后的状态（通常取 Poincaré 截面上的点）。

一条线 → 周期；两条 → 倍周期；一片密点 → **混沌**。

**它直接展示"模式如何随参数转变"** —— 也就是 `mode-classification` 类任务的答案。

### 7. Poincaré 截面

高维系统降维：每次轨迹穿过某个超平面（同方向）时记一个点。

- **闭曲线** → 准周期（KAM 环）
- **散点云** → 混沌
- **有限个点** → 周期

### 8. ★ 参数空间相图 (regime map)

**这是 "under certain conditions" 那条任务的答案，也是 IYPT 报告里最值钱的一张图。**

**做法**：取 2 个（或更多）关键参数张成平面，**每个格点跑一次仿真**，按**结果的模式**上色：

- 磁铁加速器："**有效加速临界**" —— 哪片区域能加速、哪片不能
- 磁力牛顿摆："**$\lambda$ vs $(d, B)$**" —— 哪片区域混沌

**它直接回答"什么时候会、什么时候不会"，而这正是题面用一个介词短语藏起来的那条任务。**

> **注意**：格点扫描很贵（每个点一次完整仿真）。**先用粗网格定位边界，再在边界附近加密。**
> 而且**边界本身要有物理解释** —— "临界线大约是 $\Pi = 1$" 比 "这里有条线" 值钱得多。

### 9. 吸引域 (basin of attraction)

多稳态系统：初始条件平面，按"最终落到哪个吸引子"上色。

**分形边界 = 对初值极端敏感**，这本身就是一个结论。

## ★ 混沌的**机制分解** —— 不只是"它混沌了"

**"它是混沌的"是观察；"它为什么混沌"才是物理。**

真实做法（磁力牛顿摆）：把运动方程**逐项摆出来**，问**哪一项负责混沌**：

> 惯性项 / 阻尼项 / 重力项 / **磁力项** → 磁力项贡献**非线性耦合**
>
> 混沌需要**三个条件同时满足**：
> 1. **耦合通道足够多**（摆数 ≥ 3）
> 2. **磁力项与重力项可比拟**（否则重力主导，退化成一堆独立单摆）
> 3. **初始角足够大**（激活非线性）

**这三条是可以被实验和仿真分别证伪的。**

> 这和 Skill 1 的**机制预算**是同一个动作，只是搬到了动力学上：
> **每一项都要给数字，并说清它是不是那个负责人。**

## 完成的标准

不是"图很多"，是这三件事都做到了：

1. **每条 `task` 都被至少一张图或一个 target 回答了**，并且能逐条打勾
2. **每条 `model_validation_check` 都跑了** —— 中间量被独立验证，不是靠"最终结果对了"倒推
3. **`essence.one_sentence` 里的每个关键词，都有一张对应的图** —— 说了"非线性"就得有频谱/相图；说了"非对称转换"就得有势能景观

## 常见的自我背叛

- **只算了 `targets[]` 里那几个数，图也画了，然后收工。** —— 你答了任务，但**没验证模型、也没阐释本质**。分数在后两样。
- **用最终结果的吻合，来证明中间环节都对。** —— 两个错误可以抵消。**去测中间量。**
- **只画了一条相图轨迹。** 相图的信息在轨迹**之间**。
- **算了 Lyapunov 但没做"可积系统 λ→0"的对拍。** 那你不知道你算的是混沌还是浮点噪声。
- **报告"系统在某些参数下混沌"，但没画参数空间相图。** "某些"是多少？**那正是题目问的。**
- **说了"它混沌"，但没说"哪一项负责"。** 观察 ≠ 物理。
