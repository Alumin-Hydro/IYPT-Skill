# 验证阶梯：六道门

**严格按序。前一道不过，不许走下一道。**

这份清单的存在理由只有一个：**当曲线和预期不符时，你必须能说出"是代码错还是物理错"。** 说不出来，你就会去改图——不是因为你不诚实，是因为你**没有别的动作可做**。

---

## Gate 0 · 极限对拍（Limit Cross-Check）

**这是整个阶梯的地基。先跑它，跑不过一律不许往下走。**

### 它是什么

`equations[].numerical_notes` 里写好的配方：**数值解在某个极限下必须回到闭式解**。

magnetic-brake 的例子：完整的有限长磁体 + 有限壁厚积分式 (15)，在「整个磁体缩小 + 薄壁」的极限下，必须回到点偶极子闭式解 (10)：

$$b \;\xrightarrow[\;R,L\to 0,\ m\ \text{固定};\ w\to 0\;]{}\; \frac{45}{1024}\frac{\mu_0^2 m^2\sigma w}{a^4}$$

### 为什么它是地基

**它是纯数学恒等式，与物理对错无关。**

即使 Skill 1 的整个物理模型都错了——比如它漏掉了一个完全不同的机制——(15) 在那个极限下**仍然**必须回到 (10)，因为这是同一个积分在两种参数下的值，是数学，不是物理。

于是：

- **Gate 0 不过 → 一定是代码错。** 不用想别的。
- **Gate 0 过了 → 代码被证明是对的。** 此后任何不符，都指向物理。

**没有 Gate 0，后面每一个不符都是糊涂账。** 你永远不知道该改代码还是改模型，于是你会改图。

### 怎么跑

**扫一串 $\epsilon$，看误差是否单调趋于 0。**

```
   eps      b_model2       比值        误差
  1.000   1.505536e-04   0.714613   28.5387%
  0.300   2.054489e-04   0.975177    2.4823%
  0.100   2.101326e-04   0.997408    0.2592%
  0.030   2.106235e-04   0.999739    0.0261%
  0.010   2.106662e-04   0.999941    0.0059%   <- 通过（门槛 0.1%）
```

**只取一个点，你分不清"通过了"和"碰巧"。**

误差应当以某个幂次趋于 0（这里是 $O(\epsilon^2)$：$\epsilon$ 缩 3× 误差缩 ~10×）。**误差不降、或降到某个值就不动了 → 那个残留值就是你的 bug 的大小。**

### 先在纸上验配方本身

> **血泪教训（真实发生）**：magnetic-brake 的原始配方写的是"令 $L\to0$ 且保持 $m = M_s V$ 固定（退化为点偶极子）、同时令 $w\to0$"。
>
> 照做，比值收敛到 **3.5505，不是 1**：
>
> ```
>    L (mm)     b_model2/b_model0
>   10.0000          0.7146
>    1.0000          3.3394
>    0.1000          3.5484
>    0.0300          3.5505    <- 收敛了，但收敛到 3.55
> ```
>
> **原因**：只让 $L\to0$ 而 $R$ 保持 5 mm，得到的是一个**半径 5 mm 的薄圆盘**。在 $r=a=6$ mm 处它根本不是点偶极子——模数 $k^2 = 4Ra/(R+a)^2 = 0.99$，离远场十万八千里。点偶极子极限要求**整个磁体**相对 $a$ 缩小：$R$ 和 $L$ 必须**一起**趋于 0（$R\to\epsilon R,\ L\to\epsilon L,\ M_s\to M_s/\epsilon^3$，使 $m = M_s\pi R^2 L$ 不变）。
>
> 补上 $R\to0$，误差立刻掉到 0.0059%。

**"收敛到一个稳定值、但不是 1"是「极限存在，但取错了极限」的指纹。** 认得出这个指纹，你就知道该报 `SPEC-DEFECT`，而不是去改代码。

### 没有极限对拍配方怎么办

`numerical_notes` 里没写 → 这本身就是 `SPEC-DEFECT`。**但你通常能自己造一个**：

- 某个参数 → 0 或 → ∞ 时退化成教科书结果？
- 关掉某个机制（令其系数为 0）时回到 Model-0？
- 网格 → ∞ 细时逼近解析解？
- 有解析特解（平衡态、定常解、自相似解）可以代入验证？

**造不出任何一个极限对拍的模型是危险的**——它意味着你没有任何独立手段能证明代码是对的。这一点要在报告里说出来。

## Gate 1 · 收敛门

**结果必须与数值参数无关。** 三件事各做一遍：

| 拧什么 | 拧多少 |
|---|---|
| 求解器 / 求积容差 | 收紧 10×（`rtol: 1e-8 → 1e-9`） |
| 求积 / 差分网格 | 加密 2× |
| 广义积分的截断范围 | 扩大 2× |

**结果变化必须 < 断言容差的 1/10。**

> **不做这个，你那个"斜率 −1.02"可能纯粹是网格太粗。** 然后你会以为自己发现了物理。

截断范围尤其阴——$\int_{-\infty}^{\infty}$ 你总得在某处截断。截短了，结果**系统性偏小**，而且偏得很平滑，看不出来。**唯一的办法是把截断范围翻倍再算一遍。**

### ★ 必须在**扫描端点**上也做，不能只在基准点做

**这和 Skill 1 的「机制预算必须做扫描端点检查」是同一个道理**：基准点收敛，**不代表**扫描的另一端也收敛。

> **血泪教训（冒烟测试注入 4，真实复现）**：把广义积分的截断长度写成**绝对值**（$z_{\max} = 24$ mm），而不是自然尺度的倍数。
>
> - 在基准点 $a = 6$ mm 处，$z_{\max} = 4.0a$ → 误差 0.026%，**Gate 0 也过得去**（0.0171%）
> - 在扫描上端 $a = 12$ mm 处，$z_{\max}$ 只剩 $2.0a$ → 误差 **1.05%**
>
> 结果：曲线光滑、单调、两个 `must_not` 都不触发、只在基准点检查的收敛门也过。**大 $a$ 端的每个点都偏小一点点，于是拟合出的斜率是 $k = 3.4509$，而真值是 $3.4392$——差 0.012，肉眼完全看不出来。**
>
> **只有「在扫描端点上也检查收敛」的 Gate 1 抓到了它**（截断翻倍后变化 1.04%）。

**截断长度永远写成自然尺度的倍数**（`zmax = 200 * a`），不要写绝对值——**参数扫描时那个自然尺度会变**。

## Gate 2 · 分层对拍

**Model-0 → Model-1 → Model-2，每层算完立刻和上一层在重叠区对拍。**

- **Model-0 必须精确重现 `targets[].baseline_value`。** 重现不了，是参数读错了或单位错了——**先修这个，别往下走**。
- Model-1 关掉修正项 → 必须回到 Model-0
- Model-2 在 Model-1 的适用域内 → 应当接近 Model-1

**上来就写 Model-2，你会调三天参数也不知道哪里错了。**

## Gate 3 · 解析对拍

每个 `targets[].analytical_prediction` 都算出数值解与解析预测的**相对偏差**，写进 `results.json`。

**偏差大不一定是错。** Model-2 与 Model-0 的差距，正是某条 RISKY 假设崩溃的**量度**——那可能恰恰是本题最有价值的结果。

magnetic-brake：Model-2 的 $v_t = 5.01$ cm/s，Model-0 是 2.74 cm/s，差 83%。这**不是 bug**——这是 A-1（点偶极子近似）在 $a/L = 0.6$ 时崩溃的定量后果，而 Skill 1 早就把 A-1 标成 `RISKY` 并预言了这件事。

**但必须记录，并解释这个偏差来自哪条假设。** 记录了、解释了，它是结论；没记录，它是隐患。

> **量纲错会在这里以数量级偏差的形式暴露**，所以不需要单独的量纲门（也就不需要 sympy / pint）。

### ★★ 伪对拍：两个数看着吻合，但它们测的**不是同一个量**

**这是验证工作最阴的一种失败：它长得和"验证成功"一模一样。**

> **血泪教训（真实发生，而且是差点交付出去的）**：作者算了一张"点偶极子近似的误差 vs $a/L$"的图，在本实验的 $a/L = 0.60$ 处读出 **40%**。然后在图上写了一句：
>
> > *"and indeed $v_t$ came out +82.7%"* —— **宣布与已测到的 $v_t$ 偏差吻合。**
>
> **40% 和 82.7% 差着一倍，根本不吻合。** 作者只是"觉得都是同一回事"就写下去了。
>
> **根因**：那张图**全程用薄壁近似**，隔离的是 **A-1 一条假设**；而 82.7% 是 Model-2 vs Model-0，含 **A-1 和 A-2 两条**。**它们本来就不该相等。**
>
> 真相（事后核出来的）：$\underbrace{1.3993}_{\text{A-1}} \times \underbrace{1.3059}_{\text{A-2}} = 1.8273 \Rightarrow +82.7\%$ —— **两条假设的失效是「相乘」的，不是相加的。** 一旦分解正确，两个数完美自洽，而且**这个分解本身成了一次独立的对拍**。

**规矩：宣布"两个数吻合"之前，先问三句话。**

1. **它们真的是同一个量吗？**（同一个模型？同一组假设？同一个参数点？）
2. **如果我在"隔离某条假设"，那隔离出来的效应，本来就不该等于总效应。**
3. **多条假设的失效通常是「相乘」的，不是相加的。** 报告某条假设的贡献时，必须说清**隔离了什么、没隔离什么**。

> **数字对不上的时候，你有两条路**：一条是回头查，另一条是含糊过去。
> **含糊过去的那条路，读起来和"验证成功"一模一样——这就是它危险的原因。**

**这一条 `check_sim.py` 查不了。** 它只能靠你在写下"吻合"两个字之前，问那三句话。

## Gate 4 · `expected_shape` 断言

`acceptance.py` 全跑，逐条给 `measured` 和 `verdict`。

**不许有"未评估"的断言。跳过一条 = 契约违约。**

### 双向性：**"符得太好"也是失败**

看这条真实的 `expected_shape`：

> "Model-0 是斜率恰为 +4.00 的直线。**Model-2 预期斜率显著小于 4**。**若 Model-2 也给出 4.00，说明有限尺寸修正没被正确实现——回去查代码**"

Skill 1 预先埋了陷阱。**Model-2 若算出 4.00，那不是"验证成功"，那是修正项根本没进代码。**

**每条 `expected_shape` 都要问一遍：有没有一个"漂亮得可疑"的结果？有的话，把它写成 `must_not` 断言。**

实测：Model-0 拟合出 **4.0000**（精确），Model-2 拟合出 **3.4392**。`must_not` 未触发，F-2 通过。

### ★★ `must_not` 必须检查「结构」，不能只检查「拟合出来的数」

**这是整份文档里最贵的一条教训——它是被冒烟测试逼出来的，不是设计时想到的。**

> **血泪教训（冒烟测试注入 2，真实复现）**：往代码里偷偷塞进**点偶极子场**冒充有限长磁体的场，但**保留正确的径向积分**——也就是**只少了一个修正**。
>
> | 检查 | 结果 |
> |---|---|
> | Gate 0（极限对拍） | **PASS** —— 极限对拍时磁体本来就缩成偶极子了，两边一致，它**天然抓不到这个** |
> | Gate 1（收敛门） | PASS |
> | **`must_not`：斜率是不是 4.00 ± 0.10** | **PASS —— 拟合出 $k = 3.7869$** |
>
> **3.79 既不是 4.00（陷阱的落点），也不是真值 3.44——它从两条断言之间溜过去了。**
>
> **为什么**：`expected_shape` 埋的陷阱是按**"整个 Model-2 偷偷变成 Model-0"**校准的。但**真实的 bug 往往只少了一个修正**，落点就在两者之间。**一个拟合出来的指数，可以因为任何一处遗漏而落到任何地方。**

**正确的做法：加一条直接检查「场 / 解本身」的结构性断言。**

这道题里，是**涡流峰的位置**：

- 点偶极子场的峰**解析地**落在 $z = \pm a/2$：由 $\frac{d}{dz}\!\left[z(a^2+z^2)^{-5/2}\right] = 0 \Rightarrow a^2+z^2 = 5z^2$
- 有限长磁体的峰落在 $\approx L/2$（磁体端面上）
- 基准参数下两者相差 **67%**

注入之后，峰位精确落在 **3.00 mm $= a/2$** —— **一击即中。**

**峰位是场的结构性质，只有两个答案；拟合出的指数是连续的，可以落在任何地方。**

**怎么找到这类结构性断言**（问自己）：

- 有没有一个量，在"正确模型"和"退化模型"下取**离散地不同**的值？（峰位、节点数、对称性、渐近幂次、临界点位置）
- 有没有一个**解析可证**的特征，只有正确的实现才有？
- 如果我把某个修正项**整个删掉**，哪个量会**跳变**（而不是平滑漂移）？**那个量就是你要的 `must_not`。**

### 歧义必须显式判读

`expected_shape` 是自然语言，会自相矛盾。真实例子（F-1）：

> "两条直线，**斜率均为 -1.00**。**Model-2 在大 w 端应向上偏离 Model-0**"

这两句字面上打架：斜率既是 −1.00，又要偏离？

正确的判读是：**"−1.00" 是领头阶的断言，而"偏离"是同一句话里立刻给出的限定。** 拆成两条断言：

- $\sigma$ 面板：两条线斜率都 = −1.00 ± 0.02（实测：−1.0000 / −1.0000 ✅）
- $w$ 面板：Model-0 斜率 = −1.00 ± 0.02（实测 −1.0000 ✅）；Model-2 **必须**在大 $w$ 端上翘（实测 $v_2/v_0$ 从 1.48 单调升到 2.76 ✅）。且加一条 `must_not`：**若 Model-2 的 $w$ 斜率落在 −1.00 ± 0.05 内 → FAIL-CODE**（说明有限壁厚没实现）

**判读必须写进 `acceptance.md` 的 `interpretation` 字段，交给人眼审。** 不强制记录判读，agent 会挑对自己有利的那个读法——而且它自己都不会察觉。

## Gate 5 · RISKY 假设的数值验证

`risky_assumption_checks[]` **每条都必须跑**，按它自己的 `pass_criterion` 判定。

**假设没站住不是灾难。** 标出来的 RISKY + 一个验过它的数值实验 = 满分答案；藏起来的 RISKY = 被 Opponent 一击致命。

magnetic-brake 的实测：

| | 台账声称 | 实测 | |
|---|---|---|---|
| A-1（点偶极子） | $v_t\propto a^4$；预期 $k$ 明显 < 4 | $k = 3.44$，$\|k-4\| = 0.56 > 0.3$ | 触发**预注册**的降级动作 |
| A-2（薄壁） | "一阶修正 $O(w/a)\approx 17\%$"，门槛 <15% | $w=1$mm 处 **23.4%** | **假设没跨过它自己设的杆** |

A-2 这一条值得盯住：Skill 1 自己估的是 17%，自己把门槛设在 15%，实测 23.4%——**它没通过自己出的题**。这不是代码错（$w=3$mm 处偏差 49%，远大于 20%，"径向积分没实现"的陷阱没被触发），是**假设确实比预想的更脆**。

### `pass_criterion` 里可能预先注册了应对动作

> "若 $|k-4| > 0.3$，则 P5 被数值判定为不可靠，**必须在 `01-analysis.md` 的预测表中把 P5 降级并说明**。"

触发了就**照办**，标 `PRESCRIBED`。

**这不是事后合理化。** 动作是在数据出现**之前**写下的——这是预注册，是临床试验的金标准。执行一条预注册的应对，与"看到数据之后编一个应对"，是完全不同的两件事。

**但仍然要留痕**：归档 `model-spec-r{n}.json`，并在修订说明里指明触发条件的实测值。

## 判定表（写代码之前就定死）

| 被违反的断言 | 判定 | 为什么 |
|---|---|---|
| `limit`（Gate 0） | **FAIL-CODE** | 纯数学恒等式，与物理无关 |
| `must_not`（符得太好） | **FAIL-CODE** | 修正项根本没进代码 |
| 收敛门 | **FAIL-CODE** | 网格 / 容差 / 截断污染了结果 |
| Skill 1 承诺的定性形状（**Gate 0 已过**） | **FAIL-MODEL** | 代码已证明对 → 物理错了 → 反向边 |
| 命中 `pass_criterion` 的**预注册**分支 | **PRESCRIBED** | 不是失败。照它写的动作执行。 |

**`SPEC-DEFECT` 的硬门槛**（`docs/pipeline.md` §5.2）：

> **如果你必须引用仿真数字才能说明"契约写错了"，那它就不是 `SPEC-DEFECT`——它是模型被数据打脸了。**

Gate 0 的配方错，是**先在纸上推出来的**（点偶极子极限需要整个磁体缩小），数值只是确认 → 合法的 `SPEC-DEFECT`。

A-2 的偏差是 23% 不是 17%，**只能靠算才知道** → **不是** `SPEC-DEFECT`，是真实的假设失效。

**这条线不画死，`SPEC-DEFECT` 就是一个逃生舱。**
